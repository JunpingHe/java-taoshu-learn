# 第十四章：订单创建与事务

> **小王的疑问**：用户下单涉及创建订单、扣减库存、清空购物车...任何一步失败怎么办？如何保证要么全部成功，要么全部失败？

---

## 🤔 为什么需要事务？

### 真实场景：没有事务的灾难

```
某电商系统，没有使用事务：
┌─────────────────────────────────────────────────────────────┐
│  用户下单流程：                                             │
│  1. 创建订单（成功）                                        │
│  2. 扣减库存（失败！库存不足）                              │
│  3. 清空购物车（跳过，因为前面失败了）                      │
│  4. 发送通知（跳过）                                        │
│                                                             │
│  结果：                                                     │
│  - 订单已创建（用户看到订单成功）                           │
│  - 库存未扣减（可以重复购买）                               │
│  - 购物车还在（用户又买了一遍）                             │
│  - 用户买了2本！商家亏了！                                  │
│                                                             │
│  这就是数据不一致！                                        │
└─────────────────────────────────────────────────────────────┘
```

**🤔 什么是事务？**

```
事务 = 一组操作，要么全部成功，要么全部失败

类比：银行转账
步骤：
1. 从A账户扣100元
2. 给B账户加100元

如果没有事务：
步骤1成功，步骤2失败 → A的钱扣了，B没收到 → 钱消失了！

如果有事务：
步骤1成功，步骤2失败 → 全部回滚 → A的钱没扣，B没收到 → 正确！
```

### 事务的ACID特性

```
A - Atomicity（原子性）
   要么全部成功，要么全部失败
   例子：下单要么成功，要么完全没发生

C - Consistency（一致性）
   事务前后，数据满足约束
   例子：库存 >= 0，订单金额 = 商品单价 × 数量

I - Isolation（隔离性）
   多个事务互不干扰
   例子：A和B同时下单，互不影响

D - Durability（持久性）
   提交后永久保存
   例子：订单创建后，即使服务器重启，订单还在
```

```java
// entity/Order.java
package com.taoshu.entity;

import jakarta.persistence.*;
import lombok.Data;
import java.time.LocalDateTime;
import java.math.BigDecimal;

@Data
@Entity
@Table(name = "`order`")  # order是MySQL关键字，需要加反引号
public class Order {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    # 订单号（唯一，格式：TS + 时间戳 + 随机数）
    @Column(unique = true, length = 32)
    private String orderNo;

    # 用户ID
    @Column(nullable = false)
    private Long userId;

    # 订单状态（0-待支付 1-已支付 2-已发货 3-已完成 4-已取消）
    @Column(nullable = false)
    private Integer status = 0;

    # 金额信息（使用BigDecimal避免精度问题）
    @Column(nullable = false, precision = 10, scale = 2)
    private BigDecimal totalAmount;      # 商品总金额

    @Column(precision = 10, scale = 2)
    private BigDecimal shippingFee = BigDecimal.ZERO;  # 运费

    @Column(precision = 10, scale = 2)
    private BigDecimal discountAmount = BigDecimal.ZERO;  # 优惠

    @Column(nullable = false, precision = 10, scale = 2)
    private BigDecimal payAmount;        # 实付金额

    # 收货信息
    @Column(nullable = false, length = 50)
    private String receiverName;

    @Column(nullable = false, length = 20)
    private String receiverPhone;

    @Column(nullable = false, length = 200)
    private String receiverAddress;

    # 支付信息
    @Column(length = 20)
    private String paymentMethod;

    @Column
    private LocalDateTime paymentTime;

    @Column(length = 64)
    private String transactionNo;

    # 时间戳
    @Column(name = "created_at", updatable = false)
    private LocalDateTime createdAt;

    @Column(name = "updated_at")
    private LocalDateTime updatedAt;

    # 关联订单明细（一对多）
    @OneToMany(mappedBy = "order", cascade = CascadeType.ALL, orphanRemoval = true)
    private List<OrderItem> items;

    @PrePersist
    protected void onCreate() {
        createdAt = LocalDateTime.now();
        updatedAt = LocalDateTime.now();
        # 自动生成订单号
        if (orderNo == null) {
            orderNo = generateOrderNo();
        }
    }

    @PreUpdate
    protected void onUpdate() {
        updatedAt = LocalDateTime.now();
    }

    # 生成订单号
    private String generateOrderNo() {
        return "TS" + System.currentTimeMillis() +
               (int)(Math.random() * 10000);
    }
}
```

## 第二步：订单创建事务

### 问题：如何保证原子性？

#### 事务流程

```
用户下单 → 需要完成的操作：
├── 1. 验证商品（存在、在售、有库存）
├── 2. 创建订单（order表）
├── 3. 创建订单明细（order_item表）
├── 4. 扣减库存（book表）
├── 5. 清空购物车（Redis）
└── 6. 发送通知（异步）

如果任何一步失败：
→ 全部回滚
→ 就像什么都没发生
```

```java
// service/OrderService.java
package com.taoshu.service;

import com.taoshu.dto.*;
import com.taoshu.entity.*;
import com.taoshu.exception.BusinessException;
import com.taoshu.repository.*;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.data.redis.core.StringRedisTemplate;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;
import java.math.BigDecimal;
import java.time.LocalDateTime;
import java.util.ArrayList;
import java.util.List;
import java.util.Map;
import java.util.stream.Collectors;

@Service
public class OrderService {

    @Autowired
    private OrderRepository orderRepository;

    @Autowired
    private OrderItemRepository orderItemRepository;

    @Autowired
    private BookRepository bookRepository;

    @Autowired
    private CartService cartService;

    @Autowired
    private StringRedisTemplate redisTemplate;

    private static final String CART_KEY_PREFIX = "cart:user:";

    # 创建订单（事务方法）
    @Transactional(rollbackFor = Exception.class)  # 任何异常都回滚
    public OrderResponse createOrder(Long userId, CreateOrderRequest request) {
        # ========== 第1步：验证商品 ==========
        List<Long> bookIds = request.getItems().stream()
            .map(CreateOrderRequest.OrderItemRequest::getBookId)
            .collect(Collectors.toList());

        List<Book> books = bookRepository.findAllById(bookIds);
        Map<Long, Book> bookMap = books.stream()
            .collect(Collectors.toMap(Book::getId, b -> b));

        # 检查商品
        for (CreateOrderRequest.OrderItemRequest itemRequest : request.getItems()) {
            Book book = bookMap.get(itemRequest.getBookId());
            if (book == null) {
                throw new BusinessException("商品不存在：" + itemRequest.getBookId());
            }
            if (book.getIsDeleted() || book.getStatus() != 1) {
                throw new BusinessException("商品已下架：" + book.getTitle());
            }
            if (book.getStockQuantity() < itemRequest.getQuantity()) {
                throw new BusinessException("库存不足：" + book.getTitle());
            }
        }

        # ========== 第2步：计算金额 ==========
        BigDecimal totalAmount = BigDecimal.ZERO;
        for (CreateOrderRequest.OrderItemRequest itemRequest : request.getItems()) {
            Book book = bookMap.get(itemRequest.getBookId());
            BigDecimal subtotal = book.getPrice()
                .multiply(new BigDecimal(itemRequest.getQuantity()));
            totalAmount = totalAmount.add(subtotal);
        }

        BigDecimal shippingFee = calculateShippingFee(totalAmount);
        BigDecimal discountAmount = calculateDiscount(userId, totalAmount);
        BigDecimal payAmount = totalAmount.add(shippingFee).subtract(discountAmount);

        # ========== 第3步：创建订单 ==========
        Order order = new Order();
        order.setUserId(userId);
        order.setStatus(0);  # 待支付
        order.setTotalAmount(totalAmount);
        order.setShippingFee(shippingFee);
        order.setDiscountAmount(discountAmount);
        order.setPayAmount(payAmount);
        order.setReceiverName(request.getReceiverName());
        order.setReceiverPhone(request.getReceiverPhone());
        order.setReceiverAddress(request.getReceiverAddress());

        order = orderRepository.save(order);

        # ========== 第4步：创建订单明细 ==========
        List<OrderItem> orderItems = new ArrayList<>();
        for (CreateOrderRequest.OrderItemRequest itemRequest : request.getItems()) {
            Book book = bookMap.get(itemRequest.getBookId());

            OrderItem item = new OrderItem();
            item.setOrder(order);
            item.setBookId(book.getId());
            item.setBookTitle(book.getTitle());
            item.setBookAuthor(book.getAuthor());
            item.setPrice(book.getPrice());  # 快照价格
            item.setQuantity(itemRequest.getQuantity());
            item.setSubtotal(book.getPrice()
                .multiply(new BigDecimal(itemRequest.getQuantity())));

            orderItems.add(orderItemRepository.save(item));
        }

        order.setItems(orderItems);

        # ========== 第5步：扣减库存（使用乐观锁）==========
        for (CreateOrderRequest.OrderItemRequest itemRequest : request.getItems()) {
            Book book = bookMap.get(itemRequest.getBookId());

            # 使用乐观锁扣减库存
            int updated = bookRepository.decreaseStock(
                book.getId(),
                itemRequest.getQuantity(),
                book.getVersion()
            );

            if (updated == 0) {
                # 更新失败，库存已被修改
                throw new BusinessException("库存已被抢占，请重试");
            }
        }

        # ========== 第6步：清空购物车中已购买的商品 ==========
        cartService.removePurchasedItems(userId, bookIds);

        # ========== 第7步：发送通知（异步）==========
        sendOrderNotification(userId, order.getOrderNo());

        # 返回订单信息
        return OrderResponse.fromEntity(order);
    }

    # 计算运费
    private BigDecimal calculateShippingFee(BigDecimal totalAmount) {
        # 满49包邮
        if (totalAmount.compareTo(new BigDecimal("49")) >= 0) {
            return BigDecimal.ZERO;
        }
        return new BigDecimal("6");  # 6元运费
    }

    # 计算优惠
    private BigDecimal calculateDiscount(Long userId, BigDecimal totalAmount) {
        # TODO: 实现优惠券逻辑
        return BigDecimal.ZERO;
    }

    # 发送订单通知（异步）
    @Async
    public void sendOrderNotification(Long userId, String orderNo) {
        # TODO: 发送短信/邮件/推送
        System.out.println("发送订单通知：userId=" + userId + ", orderNo=" + orderNo);
    }
}
```

**@Transactional 注解详解**

```java
// 事务属性配置
@Transactional(
    rollbackFor = Exception.class,  # 任何异常都回滚
    timeout = 30,                   # 超时时间（秒）
    isolation = Isolation.READ_COMMITTED,  # 隔离级别
    propagation = Propagation.REQUIRED  # 传播行为
)

# rollbackFor：哪些异常触发回滚
- 默认：只回滚 RuntimeException 和 Error
- 建议：rollbackFor = Exception.class（所有异常都回滚）

# isolation：隔离级别
- DEFAULT：使用数据库默认级别
- READ_UNCOMMITTED：读未提交（脏读）
- READ_COMMITTED：读已提交（Oracle默认）
- REPEATABLE_READ：可重复读（MySQL默认）
- SERIALIZABLE：串行化（最严格，最慢）

# propagation：传播行为
- REQUIRED：如果已有事务，加入；否则创建新事务（默认）
- REQUIRES_NEW：总是创建新事务
- SUPPORTS：如果已有事务，加入；否则非事务执行
- NOT_SUPPORTED：总是非事务执行
- NEVER：如果有事务，抛异常
- MANDATORY：必须有事务，否则抛异常
- NESTED：嵌套事务（回滚只影响当前事务）
```

## 第三步：库存扣减（乐观锁）

### 问题：如何防止超卖？

#### 超卖场景

```
商品库存：1本
用户A下单 → 检查库存：1本（通过）
用户B下单 → 检查库存：1本（通过）
用户A扣库存 → 库存：0本
用户B扣库存 → 库存：-1本（超卖！）

问题：两个用户同时买到库存不足的商品
```

### 解决方案对比

```
方案1：悲观锁（SELECT FOR UPDATE）
SELECT * FROM book WHERE id = 123 FOR UPDATE;
UPDATE book SET stock = stock - 1 WHERE id = 123;

优点：简单，保证绝对安全
缺点：并发性能差（排队等待）

方案2：乐观锁（版本号） - 推荐！
UPDATE book
SET stock = stock - 1, version = version + 1
WHERE id = 123 AND version = 5;

优点：并发性能好
缺点：需要重试机制

淘书网选择：乐观锁
```

```java
// repository/BookRepository.java
@Repository
public interface BookRepository extends JpaRepository<Book, Long> {

    /**
     * 扣减库存（乐观锁）
     * @param bookId 商品ID
     * @param quantity 扣减数量
     * @param version 版本号
     * @return 影响行数（0=失败，1=成功）
     */
    @Modifying
    @Query("UPDATE Book b " +
           "SET b.stockQuantity = b.stockQuantity - :quantity, " +
           "    b.soldQuantity = b.soldQuantity + :quantity, " +
           "    b.version = b.version + 1 " +
           "WHERE b.id = :bookId " +
           "  AND b.version = :version " +
           "  AND b.stockQuantity >= :quantity")
    int decreaseStock(
        @Param("bookId") Long bookId,
        @Param("quantity") Integer quantity,
        @Param("version") Integer version
    );
}
```

## 第四步：订单状态管理

### 问题：如何管理订单状态流转？

#### 订单状态机

```
                    ┌─────────────┐
                    │   待支付    │ (0)
                    └──────┬──────┘
                           │ 支付/超时取消
                  ┌────────┴────────┐
                  ↓                 ↓
           ┌─────────────┐   ┌─────────────┐
           │   已支付    │   │   已取消    │ (4)
           │   (1)       │   └─────────────┘
           └──────┬──────┘
                  │ 发货
                  ↓
           ┌─────────────┐
           │   已发货    │ (2)
           └──────┬──────┘
                  │ 确认收货
                  ↓
           ┌─────────────┐
           │   已完成    │ (3)
           └─────────────┘

规则：
- 已支付 → 已发货：只能商家操作
- 已发货 → 已完成：只能用户操作
- 待支付 → 已取消：超时或用户取消
```

```java
# 支付订单
@Transactional(rollbackFor = Exception.class)
public void payOrder(Long userId, String orderNo, String transactionNo) {

    Order order = orderRepository.findByOrderNo(orderNo)
        .orElseThrow(() -> new BusinessException("订单不存在"));

    order.setStatus(1);  # 已支付
    order.setPaymentTime(LocalDateTime.now());
    order.setTransactionNo(transactionNo);
```

## ⚠️ 事务最佳实践

### 1. 事务范围

```java
// ❌ 错误：事务范围太大
@Transactional
public void processOrder() {
    validateOrder();      # 校验
    createOrder();        # 数据库操作
    sendSMS();            # 发短信（外部API，可能很慢）
    sendEmail();          # 发邮件（外部API，可能很慢）
    updateStatistics();   # 统计分析（可能很慢）
}
// 问题：事务持有时间长，锁竞争激烈

// ✅ 正确：事务只包含数据库操作
@Transactional
public void processOrder() {
    validateOrder();
    createOrder();
}
// 事务提交后，异步执行其他操作
@Async
public void afterOrder(Long orderId) {
    sendSMS();
    sendEmail();
    updateStatistics();
}
```

### 2. 事务边界

```java
// ❌ 错误：事务方法调用同类方法
@Service
public class OrderService {

    @Transactional
    public void createOrder() {
        saveOrder();
        updateStock();  # 不会开启新事务！
    }

    @Transactional
    public void updateStock() {
        # 这个事务注解无效
    }
}
// 原因：Spring AOP代理，同类方法调用不经过代理

// ✅ 正确：拆分Service
@Service
public class OrderService {
    @Autowired
    private StockService stockService;

    @Transactional
    public void createOrder() {
        saveOrder();
        stockService.updateStock();  # 会开启新事务
    }
}

@Service
public class StockService {
    @Transactional
    public void updateStock() {
        # 独立事务
    }
}
```

### 3. 异常处理

```java
// ❌ 错误：捕获异常不抛出
@Transactional
public void createOrder() {
    try {
        saveOrder();
    } catch (Exception e) {
        e.printStackTrace();
        # 异常被吞掉，事务不会回滚！
    }
}

// ✅ 正确：抛出异常
@Transactional(rollbackFor = Exception.class)
public void createOrder() {
    try {
        saveOrder();
    } catch (Exception e) {
        log.error("创建订单失败", e);
        throw e;  # 抛出异常，触发回滚
    }
}
```

## 📊 Django vs Spring Boot 对比

| 功能 | Django | Spring Boot | 难度 |
|------|--------|-------------|------|
| 事务声明 | `@transactional` 装饰器 | `@Transactional` 注解 | 相当 |
| 乐观锁 | `F对象 + version` | `@Version` 注解 | Spring Boot更简单 |
| 悲观锁 | `select_for_update()` | `@Lock` 或 `FOR UPDATE` | 相当 |
| 事务传播 | 无 | `propagation` 属性 | Spring Boot更强 |
| 事务隔离 | 无 | `isolation` 属性 | Spring Boot更强 |

**Django优势**：
- 装饰器语法简洁

**Spring Boot优势**：
- 传播行为更灵活
- 隔离级别可配置
- 事件驱动解耦

## 🎯 本章小结

| 知识点 | 为什么需要 | 怎么做 |
|--------|-----------|--------|
| **事务** | 保证数据一致性，要么全成功要么全失败 | @Transactional注解 |
| **ACID** | 事务的四个特性 | 原子性、一致性、隔离性、持久性 |
| **乐观锁** | 防止超卖，提高并发 | @Version + WHERE version |
| **悲观锁** | 绝对安全，性能差 | SELECT FOR UPDATE |
| **状态机** | 管理订单状态流转 | 状态枚举 + 状态检查 |
| **快照** | 保留历史数据 | 订单明细存价格快照 |

**记忆口诀**：
- **事务保一致**，**ACID记心里**
- **乐观锁防超卖**，**版本号比对**
- **悲观锁绝对**，**并发性能差**
- **状态机流转**，**快照留历史**
