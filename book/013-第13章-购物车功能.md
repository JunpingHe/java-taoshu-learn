## 🤔 为什么需要购物车？

### 真实场景：没有购物车的痛苦

```
❌ 没有购物车的电商：
用户在《Java核心技术》页面点击"立即购买"
      → 跳转到订单确认页
      → 发现还需要买《算法导论》
      → 返回搜索《算法导论》
      → 再次点击"立即购买"
      → 又想起还需要一本《设计模式》
      → 又返回搜索...

用户崩溃："为什么不能先把要买的书放一起，一起结账？"
用户离开 → 流失！
```

**购物车的价值**：
1. **用户体验**：先挑选，后结算，不用来回跳转
2. **提升客单价**：用户可以慢慢凑单，买更多
3. **减少放弃**：不用每次都填写收货信息
4. **营销机会**：购物车提醒、优惠券推送

### 🤔 数据存Redis还是数据库？

#### 方案对比

```
方案1：存数据库（MySQL）
┌─────────────────────────────────────────────────────────────┐
│  user_cart 表                                               │
│  ├─ user_id                                                │
│  ├─ book_id                                                │
│  ├─ quantity (数量)                                        │
│  └─ created_at                                             │
│                                                             │
│  优点：                                                     │
│  - 数据永久保存                                             │
│  - 支持复杂查询                                             │
│                                                             │
│  缺点：                                                     │
│  - 每次加购都要写数据库（性能差）                            │
│  - 并发加购需要锁（复杂）                                    │
│  - 数据库压力大                                             │
└─────────────────────────────────────────────────────────────┘

方案2：存Redis（推荐！）
┌─────────────────────────────────────────────────────────────┐
│  Key: cart:user:{userId}                                   │
│  Value: Hash {bookId: quantity}                            │
│                                                             │
│  优点：                                                     │
│  - 读写快（内存操作）                                       │
│  - 支持原子操作（天然线程安全）                              │
│  - 支持过期时间（购物车数据可以丢）                          │
│                                                             │
│  缺点：                                                     │
│  - 数据可能丢失（可接受，购物车丢了影响小）                  │
│  - 不支持复杂查询（购物车不需要）                            │
└─────────────────────────────────────────────────────────────┘

方案3：混合（最优！）
- Redis存购物车（快速操作）
- 下单时同步到数据库（订单表）
- 购物车数据丢失影响小，重新加购即可
```

## 第一步：数据结构设计

### Redis数据结构选择

```
问题：用什么数据结构存储购物车？

方案1：String（JSON）
key: cart:user:1
value: '{"123": 2, "456": 1}'  # {"bookId": quantity}

操作：
获取：GET cart:user:1 → JSON解析
加购：GET → 解析 → 修改 → 序列化 → SET
问题：不是原子操作，并发不安全

方案2：List
key: cart:user:1
value: [123, 123, 456]  # 每个商品出现N次

操作：
加购：LPUSH cart:user:1 123
问题：无法知道数量，需要遍历统计

方案3：Hash（推荐！）
key: cart:user:1
field: bookId
value: quantity

操作：
加购：HINCRBY cart:user:1 123 1  # 原子操作！
修改：HINCRBY cart:user:1 123 -1  # 减少数量
删除：HDEL cart:user:1 123
获取：HGETALL cart:user:1

优点：
1. 天然支持 key-value（bookId-quantity）
2. 所有操作都是原子的（并发安全）
3. 单次操作获取全部（性能好）
```

### 完整数据结构

```
# 购物车数据
cart:user:{userId}
  field: bookId
  value: {quantity}|{selected}
  示例：123 = 2|1  (bookId=123, 数量=2, 选中=1)

# 选中状态解释
selected=1: 选中（参与结算）
selected=0: 未选中（不参与结算）
```

## 第二步：加购功能

### 问题：如何处理并发加购？

#### 场景分析

```
场景：用户快速点击"加购"按钮2次

时间线：
T1: 用户点击 → quantity=1
T2: 用户点击 → quantity=?

如果是数据库：
UPDATE cart SET quantity=quantity+1 WHERE user_id=1 AND book_id=123
问题：需要行锁，性能差

如果是Redis（String）：
GET cart → 解析 → quantity+1 → SET cart
问题：T1和T2都读到quantity=1，最终quantity=2（错误！）

如果是Redis（Hash HINCRBY）：
HINCRBY cart:user:1 123 1  # 原子操作！
T1执行 → quantity=1
T2执行 → quantity=2
正确！
```

#### 2. 创建购物车Service

```java
// service/CartService.java
package com.taoshu.service;

import com.taoshu.dto.AddCartRequest;
import com.taoshu.entity.Book;
import com.taoshu.exception.BusinessException;
import com.taoshu.repository.BookRepository;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.data.redis.core.StringRedisTemplate;
import org.springframework.stereotype.Service;
import java.util.HashMap;
import java.util.Map;

@Service
public class CartService {

    @Autowired
    private StringRedisTemplate redisTemplate;

    @Autowired
    private BookRepository bookRepository;

    private static final String CART_KEY_PREFIX = "cart:user:";
    private static final int CART_EXPIRE_DAYS = 30;  # 30天过期

    # 加购
    public void addToCart(Long userId, AddCartRequest request) {
        # 1. 验证商品是否存在
        Book book = bookRepository.findById(request.getBookId())
            .orElseThrow(() -> new BusinessException("商品不存在"));

        # 2. 验证商品状态（在售、有库存）
        if (book.getStatus() != 1) {
            throw new BusinessException("商品已下架");
        }
        if (book.getStockQuantity() < request.getQuantity()) {
            throw new BusinessException("库存不足");
        }

        # 3. 加购（原子操作）
        String key = CART_KEY_PREFIX + userId;
        String field = request.getBookId().toString();
        String value = request.getQuantity() + "|1";  # quantity|selected

        # 如果商品已存在，累加数量
        Boolean exists = redisTemplate.opsForHash().hasKey(key, field);
        if (Boolean.TRUE.equals(exists)) {
            # 获取当前数量
            String currentValue = (String) redisTemplate.opsForHash().get(key, field);
            String[] parts = currentValue.split("\\|");
            int currentQuantity = Integer.parseInt(parts[0]);
            int newQuantity = currentQuantity + request.getQuantity();

            # 验证库存
            if (book.getStockQuantity() < newQuantity) {
                throw new BusinessException("库存不足，当前购物车已有" + currentQuantity + "件");
            }

            # 更新数量
            String newValue = newQuantity + "|1";
            redisTemplate.opsForHash().put(key, field, newValue);
        } else {
            # 新增商品
            redisTemplate.opsForHash().put(key, field, value);
        }

        # 4. 设置过期时间（每次操作都刷新）
        redisTemplate.expire(key, CART_EXPIRE_DAYS, TimeUnit.DAYS);
    }
}
```

**🤔 为什么设置过期时间？**

```
场景：
- 用户加购了商品
- 30天内未登录
- Redis自动清理过期数据
- 节省内存

类似：京东/淘宝购物车也会定期清理
```

## 第三步：查看购物车

### 问题：如何高效展示购物车？

#### 数据流程

```
Redis数据：
cart:user:1 = {
  "123": "2|1",  # bookId=123, quantity=2, selected=1
  "456": "1|0",  # bookId=456, quantity=1, selected=0
  "789": "3|1"   # bookId=789, quantity=3, selected=1
}

需要展示：
1. 商品详细信息（从数据库查询）
2. 购物车中的数量（从Redis获取）
3. 选中状态（从Redis获取）

流程：
1. 从Redis获取购物车所有商品ID
2. 批量查询数据库（WHERE id IN (123, 456, 789)）
3. 合并Redis数据 + 数据库数据
4. 返回给前端
```

### 代码实现：查看购物车

#### 1. 创建购物车响应DTO

```java
// dto/CartItemResponse.java
package com.taoshu.dto;

import lombok.Data;
import java.time.LocalDateTime;

@Data
public class CartItemResponse {
    private Long bookId;
    private String title;
    private String author;
    private String coverImage;
    private Integer price;
    private Integer originalPrice;
    private Integer bookCondition;
    private Integer stockQuantity;
    private Integer quantity;        # 购物车中的数量
    private Boolean selected;         # 是否选中
    private Boolean valid;            # 是否有效（商品存在、在售、有库存）

    # 计算字段
    public Integer getSubtotal() {
        return valid ? price * quantity : 0;
    }
}

// dto/CartResponse.java
@Data
public class CartResponse {
    private List<CartItemResponse> items;
    private Integer totalCount;       # 总商品数
    private Integer selectedCount;    # 选中商品数
    private Integer totalAmount;      # 总金额

    public void calculate() {
        totalCount = items.stream().mapToInt(CartItemResponse::getQuantity).sum();
        selectedCount = items.stream().filter(CartItemResponse::getSelected).mapToInt(CartItemResponse::getQuantity).sum();
        totalAmount = items.stream()
            .filter(item -> item.getValid() && item.getSelected())
            .mapToInt(CartItemResponse::getSubtotal).sum();
    }
}
```

#### 2. 在Service中实现查询逻辑

```java
// service/CartService.java
public CartResponse getCart(Long userId) {
    # 1. 从Redis获取购物车
    String key = CART_KEY_PREFIX + userId;
    Map<Object, Object> rawCart = redisTemplate.opsForHash().entries(key);

    if (rawCart.isEmpty()) {
        CartResponse response = new CartResponse();
        response.setItems(new ArrayList<>());
        response.calculate();
        return response;
    }

    # 2. 提取所有商品ID
    List<Long> bookIds = rawCart.keySet().stream()
        .map(key -> Long.parseLong(key.toString()))
        .collect(Collectors.toList());

    # 3. 批量查询数据库
    List<Book> books = bookRepository.findAllById(bookIds);
    Map<Long, Book> bookMap = books.stream()
        .collect(Collectors.toMap(Book::getId, b -> b));

    # 4. 构建响应
    List<CartItemResponse> items = new ArrayList<>();
    for (Map.Entry<Object, Object> entry : rawCart.entrySet()) {
        Long bookId = Long.parseLong(entry.getKey().toString());
        String value = (String) entry.getValue();
        String[] parts = value.split("\\|");
        Integer quantity = Integer.parseInt(parts[0]);
        Boolean selected = parts[1].equals("1");

        CartItemResponse item = new CartItemResponse();
        item.setBookId(bookId);
        item.setQuantity(quantity);
        item.setSelected(selected);

        Book book = bookMap.get(bookId);
        if (book != null && !book.getIsDeleted() && book.getStatus() == 1) {
            # 商品有效
            item.setValid(true);
            item.setTitle(book.getTitle());
            item.setAuthor(book.getAuthor());
            item.setCoverImage(book.getCoverImage());
            item.setPrice(book.getPrice());
            item.setOriginalPrice(book.getOriginalPrice());
            item.setBookCondition(book.getBookCondition());
            item.setStockQuantity(book.getStockQuantity());

            # 检查库存
            if (book.getStockQuantity() < quantity) {
                item.setValid(false);
            }
        } else {
            # 商品无效（下架、删除）
            item.setValid(false);
            item.setTitle("商品已下架");
        }

        items.add(item);
    }

    # 5. 计算总计
    CartResponse response = new CartResponse();
    response.setItems(items);
    response.calculate();

    return response;
}
```

**🤔 为什么批量查询？**

```
❌ N+1查询（慢）：
for (Long bookId : bookIds) {
    Book book = bookRepository.findById(bookId);  # N次查询
}

✅ 批量查询（快）：
List<Book> books = bookRepository.findAllById(bookIds);  # 1次查询
SQL: SELECT * FROM book WHERE id IN (123, 456, 789)

性能对比：
100个商品
- N+1查询：100次数据库交互 → 5000ms
- 批量查询：1次数据库交互 → 50ms
速度提升：100倍！
```

## 第四步：修改数量

### 问题：如何安全地修改数量？

#### 场景分析

```
场景：用户修改购物车中某商品数量

输入：
bookId = 123
newQuantity = 5

验证：
1. 商品是否存在
2. 商品是否在售
3. 库存是否充足

- quantity = 0：相当于删除
- quantity < 0：非法输入
- quantity > 99：限制最大数量
```

### 代码实现：修改数量

```java
// service/CartService.java
public void updateQuantity(Long userId, Long bookId, Integer quantity) {
    # 1. 参数校验
    if (quantity < 0) {
        throw new BusinessException("数量不能为负数");
    }
    if (quantity > 99) {
        throw new BusinessException("数量不能超过99");
    }

    # 2. 查询商品
    Book book = bookRepository.findById(bookId)
        .orElseThrow(() -> new BusinessException("商品不存在"));

    # 3. 验证库存
    if (book.getStockQuantity() < quantity) {
        throw new BusinessException("库存不足");
    }

    String key = CART_KEY_PREFIX + userId;
    String field = bookId.toString();

    # 4. 更新数量
    if (quantity == 0) {
        # 数量为0，删除商品
        redisTemplate.opsForHash().delete(key, field);
    } else {
        # 获取当前选中状态
        String currentValue = (String) redisTemplate.opsForHash().get(key, field);
        if (currentValue != null) {
            String[] parts = currentValue.split("\\|");
            Boolean selected = parts[1].equals("1");

            # 保持选中状态，更新数量
            String newValue = quantity + "|" + (selected ? "1" : "0");
            redisTemplate.opsForHash().put(key, field, newValue);
        } else {
            throw new BusinessException("购物车中没有该商品");
        }
    }

    # 5. 刷新过期时间
    redisTemplate.expire(key, CART_EXPIRE_DAYS, TimeUnit.DAYS);
}
```

public void toggleSelected(Long userId, Long bookId) {
    String key = CART_KEY_PREFIX + userId;

    # 获取当前值
    String currentValue = (String) redisTemplate.opsForHash().get(key, field);

## ⚠️ 并发安全问题

### 问题：Redis原子操作够吗？

#### 场景：加购 + 库存扣减

```
时间线：
T1: 用户加购（库存检查通过）
T2: 另一个用户买走最后一件（库存变为0）
T3: 用户下单 → 库存不足！

问题：
购物车检查库存 ≠ 下单时的库存
中间有时间差！
```

#### 解决方案：乐观锁

```java
// 下单时再次检查库存
@Transactional
public Order createOrder(Long userId, CreateOrderRequest request) {
    for (OrderItem item : request.getItems()) {
        Book book = bookRepository.findById(item.getBookId())
            .orElseThrow(() -> new BusinessException("商品不存在"));

        # 使用版本号（乐观锁）
        if (book.getStockQuantity() < item.getQuantity()) {
            throw new BusinessException("库存不足：" + book.getTitle());
        }

        # 扣减库存（带版本号检查）
        int updated = bookRepository.decreaseStock(
            item.getBookId(),
            item.getQuantity(),
            book.getVersion()  # 版本号
        );

        if (updated == 0) {
            # 更新失败，库存已被修改
            throw new BusinessException("库存已被抢占，请重试");
        }
    }

    # 创建订单...
}
```

```sql
-- BookRepository中的SQL（乐观锁）
UPDATE book
SET stock_quantity = stock_quantity - #{quantity},
    version = version + 1
WHERE id = #{bookId}
  AND version = #{version}
  AND stock_quantity >= #{quantity}
```

## 📊 Django vs Spring Boot 对比

| 功能 | Django | Spring Boot | 难度 |
|------|--------|-------------|------|
| 购物车存储 | Session / 数据库 | Redis | 相当 |
| 并发安全 | 行锁 / 乐观锁 | Redis原子操作 / 乐观锁 | 相当 |
| 批量查询 | `id__in=[...]` | `findAllById([...])` | 相当 |
| 过期时间 | Session配置 | Redis TTL | 相当 |

**Django优势**：
- Session开箱即用

**Spring Boot优势**：
- Redis性能更好
- 支持分布式部署
- 原子操作更简单
