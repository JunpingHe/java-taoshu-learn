## 🤔 为什么需要第三方支付？

### 真实场景：自己对接银行的痛苦

```
某电商平台决定自己对接银行支付：
┌─────────────────────────────────────────────────────────────┐
│  需要做的事：                                               │
│  1. 与每家银行谈合作（工商、建设、农业、招商...）           │
│  2. 对接每家银行的API（接口各不相同）                       │
│  3. 申请支付牌照（金融许可）                                │
│  4. 遵守支付监管（反洗钱、实名认证）                         │
│  5. 保证资金安全（PCI DSS认证）                             │
│  6. 处理退款、对账、清算...                                 │
│                                                             │
│  成本：                                                     │
│  - 人力：20人团队                                           │
│  - 时间：1-2年                                              │
│  - 费用：数百万                                             │
│  - 风险：极高（金融监管）                                   │
│                                                             │
│  结果：                                                     │
│  项目失败，转而使用第三方支付                               │
└─────────────────────────────────────────────────────────────┘
```

**第三方支付的价值**：
1. **快速接入**：几小时完成对接，几个月变为几小时
2. **覆盖全面**：支持所有银行卡、支付宝、微信
3. **安全可靠**：央行监管、资金保障
4. **费用合理**：手续费0.6% - 1.2%
5. **体验好**：用户熟悉支付宝/微信操作

**淘书网选择**：支付宝 + 微信支付双渠道

## 第一步：支付流程理解

### 支付流程图

```
┌─────────────────────────────────────────────────────────────┐
│                   支付宝支付流程                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  用户下单                                                    │
│      ↓                                                      │
│  商户系统（淘书网）                                         │
│      ├─ 创建订单                                            │
│      ├─ 调用支付宝API                                       │
│      └─ 生成支付URL                                         │
│      ↓                                                      │
│  跳转到支付宝收银台                                         │
│      ├─ 用户登录支付宝                                     │
│      ├─ 选择支付方式                                       │
│      └─ 确认支付                                           │
│      ↓                                                      │
│  支付宝处理                                                 │
│      ├─ 扣款                                                │
│      ├─ 更新订单状态                                       │
│      └─ 发送异步通知                                       │
│      ↓                                                      │
│  商户系统接收回调                                           │
│      ├─ 验证签名（安全）                                    │
│      ├─ 更新订单状态                                       │
│      └─ 发货/通知用户                                      │
│      ↓                                                      │
│  用户查看订单 → 已支付                                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 关键概念

```
1. 同步跳转（return_url）
   - 支付成功后，浏览器跳转回商户页面
   - 用于展示支付结果给用户
   - 不可靠（用户可能关闭浏览器）

2. 异步通知（notify_url）
   - 支付宝服务器主动调用商户接口
   - 用于更新订单状态
   - 可靠（支付宝会重试7次）
   - 必须验证签名！

3. 签名验证
   - 防止伪造通知
   - 使用支付宝公钥验证签名
   - 最关键的安全措施！
```

**🤔 为什么需要公钥私钥？**

```
非对称加密原理：

商户私钥（保密）：
- 用于生成签名
- 只有商户知道

支付宝公钥（公开）：
- 用于验证签名
- 支付宝提供

流程：
1. 商户用私钥对请求签名
2. 支付宝用商户公钥验证请求
3. 支付宝用支付宝私钥对响应签名
4. 商户用支付宝公钥验证响应

类比：
- 私钥 = 你的印章（只能自己盖）
- 公钥 = 印章样本（别人可以验证真伪）
```

#### 4. 支付宝工具类

```java
// util/AlipayUtil.java
package com.taoshu.util;

import com.alipay.api.AlipayApiException;
import com.alipay.api.AlipayClient;
import com.alipay.api.DefaultAlipayClient;
import com.alipay.api.request.AlipayTradePagePayRequest;
import com.alipay.api.request.AlipayTradeQueryRequest;
import com.alipay.api.request.AlipayTradeRefundRequest;
import com.alipay.api.response.AlipayTradePagePayResponse;
import com.alipay.api.response.AlipayTradeQueryResponse;
import com.alipay.api.response.AlipayTradeRefundResponse;
import com.taoshu.config.AlipayConfig;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Component;

import java.math.BigDecimal;

@Component
public class AlipayUtil {

    @Autowired
    private AlipayConfig alipayConfig;

    private AlipayClient alipayClient;

    @PostConstruct
    public void init() {
        alipayClient = new DefaultAlipayClient(
            alipayConfig.getGateway(),
            alipayConfig.getAppId(),
            alipayConfig.getPrivateKey(),
            "json",
            "UTF-8",
            alipayConfig.getPublicKey(),
            "RSA2"
        );
    }

    /**
     * 创建支付订单
     */
    public String createPayment(String orderNo, BigDecimal amount, String subject) {
        AlipayTradePagePayRequest request = new AlipayTradePagePayRequest();
        request.setReturnUrl(alipayConfig.getReturnUrl());
        request.setNotifyUrl(alipayConfig.getNotifyUrl());

        # 构建业务参数
        request.setBizContent("{\"out_trade_no\":\"" + orderNo + "\","
            + "\"total_amount\":\"" + amount + "\","
            + "\"subject\":\"" + subject + "\","
            + "\"product_code\":\"FAST_INSTANT_TRADE_PAY\"}");

        try {
            AlipayTradePagePayResponse response = alipayClient.pageExecute(request);
            if (response.isSuccess()) {
                # 返回支付URL（跳转到支付宝）
                return response.getBody();
            } else {
                throw new RuntimeException("支付宝支付失败：" + response.getSubMsg());
            }
        } catch (AlipayApiException e) {
            throw new RuntimeException("支付宝支付失败", e);
        }
    }

    /**
     * 查询订单状态
     */
    public boolean queryPayment(String orderNo) {
        AlipayTradeQueryRequest request = new AlipayTradeQueryRequest();
        request.setBizContent("{\"out_trade_no\":\"" + orderNo + "\"}");

        try {
            AlipayTradeQueryResponse response = alipayClient.execute(request);
            return response.isSuccess() &&
                   "TRADE_SUCCESS".equals(response.getTradeStatus());
        } catch (AlipayApiException e) {
            throw new RuntimeException("查询支付状态失败", e);
        }
    }

    /**
     * 退款
     */
    public boolean refund(String orderNo, BigDecimal amount, String refundReason) {
        AlipayTradeRefundRequest request = new AlipayTradeRefundRequest();
        request.setBizContent("{\"out_trade_no\":\"" + orderNo + "\","
            + "\"refund_amount\":\"" + amount + "\","
            + "\"refund_reason\":\"" + refundReason + "\"}");

        try {
            AlipayTradeRefundResponse response = alipayClient.execute(request);
            return response.isSuccess();
        } catch (AlipayApiException e) {
            throw new RuntimeException("退款失败", e);
        }
    }
}
```

```java
// controller/PaymentController.java
package com.taoshu.controller;

import com.taoshu.common.Result;
import com.taoshu.service.PaymentService;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.web.bind.annotation.*;

import java.math.BigDecimal;
import java.util.Map;

@RestController
@RequestMapping("/api/payment")
public class PaymentController {

    @Autowired
    private PaymentService paymentService;

    # 创建支付订单
    @PostMapping("/create")
    public Result<Map<String, String>> createPayment(
            @RequestParam String orderNo,
            @RequestParam BigDecimal amount) {

        String payUrl = paymentService.createPayment(orderNo, amount);
        return Result.success(Map.of("payUrl", payUrl));
    }

    # 支付同步跳转（用户看到）
    @GetMapping("/return")
    public String paymentReturn(
            @RequestParam String out_trade_no,
            @RequestParam String trade_no,
            @RequestParam String trade_status) {

        # 更新订单状态
        paymentService.handleReturn(out_trade_no, trade_no, trade_status);

        # 跳转到订单详情页
        return "redirect:/order/" + out_trade_no;
    }

    # 支付异步通知（支付宝服务器调用）
    @PostMapping("/notify")
    public String paymentNotify(@RequestBody Map<String, String> params) {
        # 验证签名
        if (!paymentService.verifySign(params)) {
            return "failure";  # 签名验证失败
        }

        # 处理业务逻辑
        paymentService.handleNotify(params);
        return "success";  # 必须返回"success"
    }
}
```

### Service层实现

```java
// service/PaymentService.java
package com.taoshu.service;

import com.alipay.api.AlipayApiException;
import com.alipay.api.internal.util.AlipaySignature;
import com.taoshu.config.AlipayConfig;
import com.taoshu.entity.Order;
import com.taoshu.repository.OrderRepository;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;
import java.math.BigDecimal;
import java.util.Map;

@Service
public class PaymentService {

    @Autowired
    private AlipayConfig alipayConfig;

    @Autowired
    private AlipayUtil alipayUtil;

    @Autowired
    private OrderRepository orderRepository;

    # 创建支付
    public String createPayment(String orderNo, BigDecimal amount) {
        # 验证订单
        Order order = orderRepository.findByOrderNo(orderNo)
            .orElseThrow(() -> new RuntimeException("订单不存在"));

        if (order.getStatus() != 0) {
            throw new RuntimeException("订单状态异常");
        }

        if (!order.getPayAmount().equals(amount)) {
            throw new RuntimeException("金额不匹配");
        }

        # 调用支付宝
        return alipayUtil.createPayment(
            orderNo,
            amount,
            "淘书网订单-" + orderNo
        );
    }

    # 验证签名（安全关键！）
    public boolean verifySign(Map<String, String> params) {
        try {
            return AlipaySignature.rsaCheckV1(
                params,
                alipayConfig.getPublicKey(),
                "UTF-8",
                "RSA2"
            );
        } catch (AlipayApiException e) {
            return false;
        }
    }

    # 处理同步返回
    public void handleReturn(String orderNo, String tradeNo, String tradeStatus) {
        if ("TRADE_SUCCESS".equals(tradeStatus) || "TRADE_FINISHED".equals(tradeStatus)) {
            updateOrderStatus(orderNo, tradeNo);
        }
    }

    # 处理异步通知
    public void handleNotify(Map<String, String> params) {
        String orderNo = params.get("out_trade_no");
        String tradeNo = params.get("trade_no");
        String tradeStatus = params.get("trade_status");

        if ("TRADE_SUCCESS".equals(tradeStatus) || "TRADE_FINISHED".equals(tradeStatus)) {
            updateOrderStatus(orderNo, tradeNo);
        }
    }

    # 更新订单状态
    @Transactional
    public void updateOrderStatus(String orderNo, String tradeNo) {
        Order order = orderRepository.findByOrderNo(orderNo)
            .orElseThrow(() -> new RuntimeException("订单不存在"));

        # 幂等性检查：只有待支付订单才能更新
        if (order.getStatus() != 0) {
            return;  # 已处理过，直接返回
        }

        order.setStatus(1);  # 已支付
        order.setTransactionNo(tradeNo);
        order.setPaymentTime(LocalDateTime.now());
        orderRepository.save(order);

        # 后续业务逻辑
        # 1. 通知商家发货
        # 2. 发送短信给用户
        # 3. 更新库存...
    }
}
```

**🤔 为什么需要幂等性检查？**

```
场景：
1. 支付宝发送第1次通知 → 更新订单为已支付
2. 网络延迟，支付宝认为失败
3. 支付宝发送第2次通知 → 再次更新订单

如果没有幂等性检查：
- 订单状态可能混乱
- 库存被重复扣减

有了幂等性检查：
- 第2次通知：发现status!=0，直接返回
- 订单状态正确
```

## 第五步：支付安全

### 问题：如何防止伪造支付通知？

#### 攻击场景

```
恶意用户构造POST请求：
POST /api/payment/notify
{
    "out_trade_no": "TS2023xxxxx",
    "trade_no": "2023xxxxx",
    "trade_status": "TRADE_SUCCESS",
    ...
}

如果没有验证签名：
- 系统认为支付成功
- 订单状态更新为已支付
- 用户不付款就能买到商品！

损失：100%
```

### 安全措施

#### 1. 签名验证（必须！）

```java
# 验证签名
public boolean verifySign(Map<String, String> params) {
    try {
        # 使用支付宝公钥验证
        return AlipaySignature.rsaCheckV1(
            params,
            alipayConfig.getPublicKey(),  # 支付宝公钥
            "UTF-8",
            "RSA2"
        );
    } catch (AlipayApiException e) {
        log.error("签名验证失败", e);
        return false;
    }
}
```

#### 2. 金额验证

```java
# 验证金额是否匹配
Order order = orderRepository.findByOrderNo(orderNo).orElseThrow();
BigDecimal notifyAmount = new BigDecimal(params.get("total_amount"));
if (!order.getPayAmount().equals(notifyAmount)) {
    log.error("金额不匹配：订单{}，通知金额{}，实际金额{}",
        orderNo, notifyAmount, order.getPayAmount());
    return "failure";
}
```

#### 3. 订单状态验证

```java
# 只有待支付订单才能更新
if (order.getStatus() != 0) {
    log.info("订单{}已处理，跳过", orderNo);
```


## 第六步：退款处理

### 退款流程

```
用户申请退款
      ↓
验证订单状态（已支付/已发货）
      ↓
调用退款API
      ↓
支付宝处理（1-5个工作日）
      ↓
退款回调通知
      ↓
更新订单状态
      ↓
通知用户
```

### 代码实现：退款

```java
// service/RefundService.java
@Service
public class RefundService {

    @Autowired
    private AlipayUtil alipayUtil;

    @Autowired
    private OrderRepository orderRepository;

    /**
     * 申请退款
     */
    @Transactional
    public void refund(Long orderId, String reason) {
        # 1. 查询订单
        Order order = orderRepository.findById(orderId)
            .orElseThrow(() -> new RuntimeException("订单不存在"));

        # 2. 验证订单状态
        if (order.getStatus() < 1) {
            throw new RuntimeException("订单未支付");
        }
        if (order.getStatus() == 4) {
            throw new RuntimeException("订单已退款");
        }

        # 3. 调用退款API
        boolean success = alipayUtil.refund(
            order.getOrderNo(),
            order.getPayAmount(),
            reason
        );

        if (success) {
            # 4. 更新订单状态
            order.setStatus(5);  # 退款中
            orderRepository.save(order);
        } else {
            throw new RuntimeException("退款失败");
        }
    }

    /**
     * 退款回调处理
     */
    @Transactional
    public void handleRefundNotify(Map<String, String> params) {
        String orderNo = params.get("out_trade_no");
        String refundStatus = params.get("refund_status");

        if ("REFUND_SUCCESS".equals(refundStatus)) {
            Order order = orderRepository.findByOrderNo(orderNo)
                .orElseThrow(() -> new RuntimeException("订单不存在"));

            order.setStatus(4);  # 已退款
            orderRepository.save(order);

            # 恢复库存
            restoreStock(order);
        }
    }

    private void restoreStock(Order order) {
        for (OrderItem item : order.getItems()) {
            bookRepository.increaseStock(
                item.getBookId(),
                item.getQuantity()
            );
        }
    }
}
```

```
    } catch (Exception e) {

    }
}
```

```
    List<Order> orders = orderRepository
        .findPendingOrdersBefore(LocalDateTime.now().minusMinutes(15));
```
