# 第9章：商品发布与图片上传

## 🤔 为什么商品发布这么复杂？

> **小王的疑惑**：不就是填个表单，保存到数据库吗？为什么需要这么多验证？

### 从一个垃圾数据问题说起

**场景：平台上线一个月后的数据灾难**

```
运营团队发现：
→ 有大量无效的书籍数据
→ 书名乱码："&*$#"
→ ISBN格式："123"
→ 价格为0或负数
→ 图片是二维码、病毒
→ 侵犯版权的盗版书

问题根源：
→ 发布时没有严格验证
→ 前端验证可以绕过
→ 后端验证不完整
→ 图片上传没有限制
```

**后果**：

```
✅ 数据库空间浪费（无效数据占50%+）
✅ 搜索体验差（垃圾数据干扰搜索）
✅ 用户投诉（买到盗版书、图片不符）
✅ 法律风险（侵犯版权）
✅ 性能问题（无效索引降低查询速度）
```

### 9.1 书籍信息验证

#### 🤔 为什么需要这么多验证？

**场景：每种验证防止什么问题？**

```java
@Data
public class BookCreateRequest {

    // ========== 书名验证 ==========
    @NotBlank(message = "书名不能为空")
    @Size(min = 1, max = 200, message = "书名1-200字")
    private String title;

    // 为什么？
    // → 空书名：搜索时无法识别
    // → 太长：显示排版问题
    // → 太短：可能是误操作

    // ========== 作者验证 ==========
    @Size(max = 100, message = "作者名最长100字")
    private String author;

    // 为什么？
    // → 可选字段，但限制长度

    // ========== ISBN验证 ==========
    @Pattern(regexp = "^(?:ISBN(?:-1[03])?:? )?(?=[0-9X]{10}$|(?=(?:[0-9]+[- ]){3})[- 0-9X]{13}$|97[89][0-9]{10}$|(?=(?:[0-9]+[- ]){4})[- 0-9X]{17}$)", 
             message = "ISBN格式错误")
    private String isbn;

    // 为什么？
    // → ISBN有标准格式
    // → 验证格式防止随意输入
    // → 唯一性检查在Service层做

    // ========== 分类验证 ==========
    @NotNull(message = "分类不能为空")
    private Long categoryId;

    // 为什么？
    // → 每本书必须有分类
    // → 分类必须是系统中的有效分类

    // ========== 价格验证 ==========
    @NotNull(message = "价格不能为空")
    @DecimalMin(value = "0.01", message = "价格必须大于0")
    @DecimalMax(value = "99999.99", message = "价格不能超过99999.99")
    private BigDecimal price;

    @DecimalMin(value = "0.01", message = "原价必须大于0")
    private BigDecimal originalPrice;

    // 为什么？
    // → 价格不能为0或负数（免费书可以单独标记）
    // → 价格上限防止错误输入（多打了一个0）
    // → 原价必须大于现价（逻辑检查在Service层）

    // ========== 库存验证 ==========
    @NotNull(message = "库存不能为空")
    @Min(value = 1, message = "库存至少为1")
    @Max(value = 10000, message = "库存不能超过10000")
    private Integer stockQuantity;

    // 为什么？
    // → 库存为0无法销售
    // → 上限防止错误输入（多打几个0）
}
```

### 9.2 业务逻辑验证

#### 🤔 为什么有些验证要在Service层做？

**场景：Controller验证 vs Service验证**

```
Controller层验证（@Valid）：
✅ 格式验证（手机号格式）
✅ 非空验证（必填字段）
✅ 范围验证（长度、大小）

Service层验证（业务逻辑）：
✅ ISBN唯一性检查
✅ 分类是否存在
✅ 价格逻辑（现价 ≤ 原价）
✅ 用户权限（是否是卖家）
```

**为什么不在Controller全验证？**

```
原因1：Controller不应该知道数据库
→ 检查ISBN唯一需要查数据库
→ 检查分类是否存在需要查数据库
→ 这些逻辑在Service层

原因2：验证职责分离
→ 格式验证 → @Valid
→ 业务验证 → Service

原因3：可测试性
→ Service层的验证可以单独测试
→ 不依赖Web层
```

**Service层验证实现**：

```java
@Service
@RequiredArgsConstructor
public class BookService {

    private final BookRepository bookRepository;
    private final CategoryRepository categoryRepository;
    private final UserRepository userRepository;

    @Transactional
    public Book createBook(BookCreateRequest request, Long sellerId) {

        // 1. 验证卖家是否存在
        if (!userRepository.existsById(sellerId)) {
            throw new BusinessException("用户不存在");
        }

        // 2. 验证分类是否存在
        if (!categoryRepository.existsById(request.getCategoryId())) {
            throw new BusinessException("分类不存在");
        }

        // 3. 验证ISBN唯一性（如果提供了）
        if (request.getIsbn() != null &&
            !request.getIsbn().isBlank()) {
            if (bookRepository.existsByIsbn(request.getIsbn())) {
                throw new BusinessException("该ISBN已存在");
            }
        }

        // 4. 验证价格逻辑
        if (request.getOriginalPrice() != null &&
            request.getPrice().compareTo(request.getOriginalPrice()) > 0) {
            throw new BusinessException("现价不能高于原价");
        }

        // 5. 创建Book对象
        Book book = new Book();
        book.setTitle(request.getTitle());
        book.setAuthor(request.getAuthor());
        book.setIsbn(request.getIsbn());
        book.setPublisher(request.getPublisher());
        book.setCategoryId(request.getCategoryId());
        book.setCategoryName(request.getCategoryName());  // TODO: 从分类表获取
        book.setPrice(request.getPrice());
        book.setOriginalPrice(
            request.getOriginalPrice() != null ?
            request.getOriginalPrice() : request.getPrice()
        );
        book.setStockQuantity(request.getStockQuantity());
        book.setSellerId(sellerId);

        // 6. 获取卖家昵称
        User seller = userRepository.findById(sellerId).orElseThrow();
        book.setSellerName(seller.getNickname());

        // 7. 保存
        return bookRepository.save(book);
    }
}
```

### 9.3 图片上传详解

#### 🤔 为什么上传这么复杂？

**场景：各种图片上传问题**

```
问题1：文件太大
→ 用户上传10MB的原图
→ 数据库存储压力大
→ 前端加载慢
→ 手机用户流量消耗快

问题2：格式不支持
→ 用户上传.bmp、.tiff
→ 前端无法显示
→ 需要转换

问题3：图片内容不当
→ 上传二维码、病毒、广告图
→ 用户投诉

问题4：文件名冲突
→ 多个用户上传"book.jpg"
→ 互相覆盖
```

**Step 2: 后端处理**

```java
@RestController
@RequestMapping("/api/upload")
@RequiredArgsConstructor
public class UploadController {

    @Value("${app.upload.path}")
    private String uploadPath;

    @Value("${app.upload.allowed-types}")
    private String allowedTypes;

    @PostMapping("/image")
    public Result<Map<String, String>> uploadImage(
            @RequestParam("file") MultipartFile file) {

        // 1. 文件验证
        if (file.isEmpty()) {
            return Result.badRequest("请选择文件");
        }

        // 2. 文件大小验证
        if (file.getSize() > 5 * 1024 * 1024) {
            return Result.badRequest("图片大小不能超过5MB");
        }

        // 3. 文件类型验证
        String contentType = file.getContentType();
        if (!isAllowedImageType(contentType)) {
            return Result.badRequest("只支持JPG、PNG、GIF、WebP格式");
        }

        // 4. 生成唯一文件名
        String originalFilename = file.getOriginalFilename();
        String extension = getFileExtension(originalFilename);
        String filename = UUID.randomUUID() + extension;

        // 5. 图片处理（可选）
        try {
            // 压缩图片
            BufferedImage image = ImageIO.read(file.getInputStream());
            if (image.getWidth() > 2000 || image.getHeight() > 2000) {
                image = resizeImage(image, 2000, 2000);
                // 保存压缩后的图片
                ByteArrayOutputStream baos = new ByteArrayOutputStream();
                ImageIO.write(image, getFormat(extension), baos);
                // 保存压缩后的文件
            }
        } catch (IOException e) {
            // 如果处理失败，使用原图
            log.warn("图片处理失败，使用原图", e);
        }

        // 6. 保存文件
        File dest = new File(uploadPath + filename);
        file.transferTo(dest);

        // 7. 返回URL
        String url = "/uploads/" + filename;

        Map<String, String> result = new HashMap<>();
        result.put("url", url);
        result.put("filename", filename);

        return Result.success("上传成功", result);
    }

    private boolean isAllowedImageType(String contentType) {
        return Arrays.asList("image/jpeg", "image/png", "image/gif", "image/webp")
                .contains(contentType);
    }

    private String getFileExtension(String filename) {
        int lastDot = filename.lastIndexOf('.');
        return lastDot > 0 ? filename.substring(lastDot) : "";
    }

    private String getFormat(String extension) {
        return switch (extension.toLowerCase()) {
            case ".jpg", ".jpeg" -> "jpg";
            case ".png" -> "png";
            case ".gif" -> "gif";
            case ".webp" -> "webp";
            default -> "jpg";
        };
    }

    private BufferedImage resizeImage(BufferedImage original, int maxWidth, int maxHeight) {
        int width = original.getWidth();
        int height = original.getHeight();

        // 计算缩放比例
        double widthRatio = (double) maxWidth / width;
        double heightRatio = (double) maxHeight / height;
        double ratio = Math.min(widthRatio, heightRatio);

        int newWidth = (int) (width * ratio);
        int newHeight = (int) (height * ratio);

        // 缩放
        BufferedImage resized = new BufferedImage(newWidth, newHeight, BufferedImage.TYPE_INT_RGB);
        Graphics2D g = resized.createGraphics();
        g.drawImage(original, 0, 0, newWidth, newHeight, null);
        g.dispose();

        return resized;
    }
}
```

```
→ 愿觉很好
```

**前端进度显示**：

```javascript
// 使用axios的onUploadProgress
const uploadWithProgress = async (file, onProgress) => {
  const formData = new FormData();
  formData.append('file', file);

  const response = await axios.post('/api/upload/image', formData, {
    onUploadProgress: (progressEvent) => {
      const percentCompleted = Math.round(
        (progressEvent.loaded * 100) / progressEvent.total
      );
      onProgress(percentCompleted);
    }
  });

  return response.data;
```

### 9.6 图片压缩与优化

#### 🤔 为什么要压缩图片？

**场景：图片太大的影响**

```
原始图片：5MB
→ 上传时间：10秒（4G网络）
→ 存储成本：每月100元（存储+CDN）
→ 加载时间：3秒
→ 流量消耗：5MB/次

压缩后：500KB
→ 上传时间：1秒
→ 存储成本：每月10元
→ 加载时间：0.3秒
→ 流量消耗：500KB/次

节省：
→ 上传快10倍
→ 存储便宜10倍
→ 加载快10倍
→ 流量省10倍
```

**压缩算法对比**：

| 算法 | 压缩率 | 速度 | 适用场景 |
|------|-------|------|----------|
| JPEG | 高 | 快 | 照片 |
| WebP | 很高 | 快 | 现代浏览器 |
| PNG | 无损 | 慢 | 需要透明背景 |

**实现**：

```java
// 图片处理工具类
@Component
public class ImageProcessor {

    public void compressImage(String imagePath) throws IOException {
        File inputFile = new File(imagePath);
        BufferedImage image = ImageIO.read(inputFile);

        // 获取文件大小
        long fileSize = inputFile.length();

        // 如果文件大于1MB，进行压缩
        if (fileSize > 1024 * 1024) {
            // 计算目标尺寸
            int targetSize = 800;  // 宽度或高度最大800px
            int width = image.getWidth();
            int height = image.getHeight();

            if (width > targetSize || height > targetSize) {
                // 等比例缩放
                double ratio = Math.min(
                    (double) targetSize / width,
                    (double) targetSize / height
                );
                int newWidth = (int) (width * ratio);
                int newHeight = (    int) (height * ratio);

                BufferedImage resized = new BufferedImage(newWidth, newHeight, BufferedImage.TYPE_INT_RGB);
                Graphics2D g = resized.createGraphics();
                g.drawImage(image, 0, 0, newWidth, newHeight, null);
                g.dispose();

                // 覆盖原文件
                ImageIO.write(resized, "jpg", inputFile);
            }

            // 调整JPEG质量
            if (getFormat(imagePath).equals("jpg")) {
                // 重新保存，降低质量
                BufferedImage image = ImageIO.read(inputFile);
                Iterator<ImageWriter> writers = ImageIO.getImageWritersByFormatName("jpg");
                ImageWriter writer = writers.next();

                ImageWriteParam param = writer.getDefaultWriteParam();
                param.setCompressionMode(ImageWriteParam.MODE_EXPLICIT);
                param.setCompressionQuality(0.7f);  // 质量70%

                writer.setOutput(inputFile);
                writer.write(null, new IIOImage(image), param);
            }
        }
    }

    private String getFormat(String filePath) {
        String ext = filePath.substring(filePath.lastIndexOf('.') + 1);
        return ext.toLowerCase();
    }
}
```

### 9.7 图片CDN配置

#### 🤔 为什么需要CDN？

**场景：全国用户访问**

```
没有CDN：
北京用户 → 访问北京服务器 → 20ms
上海用户 → 访问北京服务器 → 50ms
广州用户 → 访问北京服务器 → 80ms
→ 距离越远，访问越慢
→ 北京服务器负载高

有CDN：
北京用户 → 访问CDN北京节点 → 5ms
上海用户 → 访问CDN上海节点 → 5ms
广州用户 → 访问CDN广州节点 → 5ms
→ 距离近，访问快
→ 负载分散
```

### 9.8 本章小结

#### 🎯 商品发布要点

| 步骤 | 验证内容 | 防止问题 |
|------|---------|----------|
| **1.参数验证** | 格式、非空、范围 | 格式错误、空值、越界 |
| **2.业务验证** | ISBN唯一、分类存在、价格逻辑 | 重复数据、无效数据 |
| **3.图片验证** | 格式、大小、内容 | 大文件、错误格式、非法图片 |
| **4.权限验证** | 是否登录、是否卖家 | 未登录、权限不足 |
| **5.数据持久化** | 保存到数据库 | 保存失败 |

#### 💡 最佳实践

1. **多层验证**：Controller格式 + Service业务
2. **图片压缩**：减少存储和带宽
3. **唯一性检查**：防止重复数据
4. **事务管理**：创建订单时需要扣减库存
5. **错误提示**：明确告诉用户怎么改
