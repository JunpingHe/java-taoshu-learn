## 🤔 为什么需要定时任务？

### 真实场景：缺少自动化的痛苦

```
某电商平台没有定时任务：
┌─────────────────────────────────────────────────────────────┐
│  运营人员每天手动操作：                                     │
│  09:00  登录后台，导出昨天订单数据                         │
│  10:00  统计销售额、订单量                                 │
│  11:00  检查超时未支付订单，手动取消                       │
│  14:00  同步库存数据                                       │
│  15:00  生成报表发送给老板                                 │
│  16:00  清理过期优惠券                                     │
│  17:00  备份数据库                                         │
│  ...                                                       │
│                                                             │
│  问题：                                                     │
│  - 耗时：每天4小时                                         │
│  - 易错：手动操作难免失误                                 │
│  - 延迟：人不能24小时在线                                  │
│  - 成本：需要专门人员                                      │
│                                                             │
│  有定时任务：                                               │
│  - 自动化执行                                              │
│  - 准确无误                                                │
│  - 7×24小时运行                                            │
│  - 释放人力                                                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**定时任务的价值**：
1. **自动化**：解放人力，减少重复工作
2. **准确性**：机器执行，不会出错
3. **实时性**：及时处理，不依赖人工
4. **成本节约**：减少人工成本

### 问题：如何实现简单定时任务？

```
场景：
1. 每天凌晨1点统计订单
2. 每小时同步库存
3. 每30秒检查超时订单
```

### 代码实现：定时任务

```java
// task/ScheduledTasks.java
package com.taoshu.task;

import lombok.extern.slf4j.Slf4j;
import org.springframework.scheduling.annotation.Scheduled;
import org.springframework.stereotype.Component;

@Slf4j
@Component
public class ScheduledTasks {

    /**
     * 场景1：固定延迟执行
     * 上次执行完成后，等待5秒再执行
     */
    @Scheduled(fixedDelay = 5000)  # 5秒
    public void task1() {
        log.info("固定延迟任务执行：{}", LocalDateTime.now());
        # 执行任务...
    }

    /**
     * 场景2：固定频率执行
     * 无论任务执行多久，每5秒执行一次
     */
    @Scheduled(fixedRate = 5000)  # 5秒
    public void task2() {
        log.info("固定频率任务执行：{}", LocalDateTime.now());
        # 执行任务...
    }

    /**
     * 场景3：初始延迟 + 固定频率
     * 应用启动后等待10秒，然后每5秒执行一次
     */
    @Scheduled(initialDelay = 10000, fixedRate = 5000)
    public void task3() {
        log.info("初始延迟任务执行：{}", LocalDateTime.now());
        # 执行任务...
    }

    /**
     * 场景4：Cron表达式
     * 每天凌晨1点执行
     */
    @Scheduled(cron = "0 0 1 * * ?")
    public void dailyStatistics() {
        log.info("每日统计任务执行：{}", LocalDateTime.now());
        # 统计昨日订单、销售额...
    }

    /**
     * 场景5：Cron表达式
     * 每小时执行一次
     */
    @Scheduled(cron = "0 0 * * * ?")
    public void hourlySync() {
        log.info("每小时同步任务执行：{}", LocalDateTime.now());
        # 同步库存...
    }

    /**
     * 场景6：Cron表达式
     * 每30秒执行一次
     */
    @Scheduled(cron = "0/30 * * * * ?")
    public void checkTimeoutOrders() {
        log.info("检查超时订单：{}", LocalDateTime.now());
        # 检查并取消超时订单...
    }

    /**
     * 场景7：Cron表达式
     * 每周一早上8点执行
     */
    @Scheduled(cron = "0 0 8 ? * MON")
    public void weeklyReport() {
        log.info("生成周报：{}", LocalDateTime.now());
        # 生成上周报表...
    }
}
```

## 第二步：Cron表达式详解

### 问题：Cron表达式是什么？

```
Cron表达式：一种定时规则的表达式

格式：
秒 分 时 日 月 周 [年]

字段范围：
秒：0-59
分：0-59
时：0-23
日：1-31
月：1-12
周：0-7 (0和7都代表周日)
年：1970-2099（可选）

特殊符号：
* ：所有值
- ：范围
, ：枚举
/ ：间隔
? ：不指定（日和周互斥，一个用?，另一个用值）
L ：最后（日、周）
W ：工作日
# ：第几个周几

示例：
0 0 12 * * ?     每天12点
0 0/5 * * * ?    每5分钟
0 0 8 ? * MON-FRI 工作日早上8点
0 0 0 L * ?      每月最后一天0点
0 0 0 ? * 6L     每月最后一个周五0点
```

### 常用Cron表达式

```
┌─────────────────────────────────────────────────────────────┐
│              常用Cron表达式                                 │
├─────────────────────────────────────────────────────────────┤
│  表达式                      │ 说明                        │
├─────────────────────────────────────────────────────────────┤
│  0/5 * * * * ?             │ 每5秒                        │
│  0 */1 * * * ?             │ 每1分钟                      │
│  0 0 */1 * * ?             │ 每1小时                      │
│  0 0 12 * * ?              │ 每天12点                     │
│  0 0 0 * * ?               │ 每天0点                      │
│  0 0 0 * * MON             │ 每周一0点                    │
│  0 0 0 1 * ?               │ 每月1号0点                   │
│  0 0 0 L * ?               │ 每月最后一天0点              │
│  0 0 8 ? * MON-FRI         │ 工作日早上8点                │
│  0 0 0,12,18 * * ?         │ 每天0点、12点、18点          │
│  0 0/30 8-10 * * ?         │ 每天8-10点，每30分钟         │
│  0 0 0 ? * 6#3             │ 每月第3个周五0点             │
│  0 0 0 LW * ?              │ 每月最后一个工作日0点         │
└─────────────────────────────────────────────────────────────┘
```

### 任务1：超时订单自动取消

```java
// task/OrderTimeoutTask.java
package com.taoshu.task;

import com.taoshu.entity.Order;
import com.taoshu.enums.OrderStatus;
import com.taoshu.repository.OrderRepository;
import com.taoshu.service.OrderStatusService;
import lombok.extern.slf4j.Slf4j;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.scheduling.annotation.Scheduled;
import org.springframework.stereotype.Component;
import java.time.LocalDateTime;
import java.util.List;

@Slf4j
@Component
public class OrderTimeoutTask {

    @Autowired
    private OrderRepository orderRepository;

    @Autowired
    private OrderStatusService orderStatusService;

    /**
     * 取消超时未支付订单
     * 每1分钟执行一次
     */
    @Scheduled(cron = "0 */1 * * * ?")
    public void cancelTimeoutOrders() {
        log.info("开始检查超时订单");

        # 查询30分钟前创建的待支付订单
        LocalDateTime timeout = LocalDateTime.now().minusMinutes(30);
        List<Order> timeoutOrders = orderRepository
            .findByStatusAndCreatedAtBefore(OrderStatus.PENDING_PAYMENT.getCode(), timeout);

        log.info("发现{}个超时订单", timeoutOrders.size());

        for (Order order : timeoutOrders) {
            try {
                # 变更状态为已取消
                orderStatusService.changeStatus(
                    order.getId(),
                    OrderStatus.CANCELLED,
                    "SYSTEM",
                    "超时未支付自动取消"
                );

                log.info("订单{}超时取消", order.getOrderNo());

            } catch (Exception e) {
                log.error("取消订单{}失败", order.getId(), e);
            }
        }

        log.info("超时订单检查完成");
    }
}
```

#### 3. 动态任务调度器

```java
// config/DynamicSchedulerConfig.java
package com.taoshu.config;

import com.taoshu.entity.ScheduledTask;
import com.taoshu.repository.ScheduledTaskRepository;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.context.annotation.Configuration;
import org.springframework.scheduling.TaskScheduler;
import org.springframework.scheduling.annotation.SchedulingConfigurer;
import org.springframework.scheduling.config.ScheduledTaskRegistrar;
import org.springframework.scheduling.support.CronTrigger;
import jakarta.annotation.PostConstruct;
import java.util.List;
import java.util.Map;
import java.util.concurrent.ConcurrentHashMap;
import java.util.concurrent.ScheduledFuture;

@Configuration
public class DynamicSchedulerConfig implements SchedulingConfigurer {

    @Autowired
    private TaskScheduler taskScheduler;

    @Autowired
    private ScheduledTaskRepository taskRepository;

    private Map<String, ScheduledFuture<?>> taskMap = new ConcurrentHashMap<>();

    @PostConstruct
    public void init() {
        # 应用启动时，从数据库加载任务
        reloadTasks();
    }

    @Override
    public void configureTasks(ScheduledTaskRegistrar taskRegistrar) {
        taskRegistrar.setTaskScheduler(taskScheduler);
    }

    /**
     * 重新加载所有任务
     */
    public void reloadTasks() {
        # 1. 停止所有正在运行的任务
        stopAllTasks();

        # 2. 从数据库加载任务配置
        List<ScheduledTask> tasks = taskRepository.findAll();

        # 3. 启动任务
        for (ScheduledTask task : tasks) {
            if (task.getEnabled()) {
                startTask(task);
            }
        }
    }

    /**
     * 启动任务
     */
    public void startTask(ScheduledTask task) {
        try {
            # 获取任务类
            Class<?> clazz = Class.forName(task.getTaskClass());
            Object taskInstance = clazz.getDeclaredConstructor().newInstance();

            # 创建Runnable
            Runnable runnable = (Runnable) taskInstance;

            # 创建CronTrigger
            CronTrigger cronTrigger = new CronTrigger(task.getCronExpression());

            # 调度任务
            ScheduledFuture<?> future = taskScheduler.schedule(
                runnable,
                cronTrigger
            );

            # 保存到Map
            taskMap.put(task.getTaskName(), future);

            log.info("任务启动成功：{}，cron：{}", task.getTaskName(), task.getCronExpression());

        } catch (Exception e) {
            log.error("启动任务失败：{}", task.getTaskName(), e);
        }
    }

    /**
     * 停止任务
     */
    public void stopTask(String taskName) {
        ScheduledFuture<?> future = taskMap.get(taskName);
        if (future != null) {
            future.cancel(false);
            taskMap.remove(taskName);
            log.info("任务停止：{}", taskName);
        }
    }

    /**
     * 停止所有任务
     */
    public void stopAllTasks() {
        taskMap.forEach((name, future) -> future.cancel(false));
        taskMap.clear();
    }

    /**
     * 更新任务
     */
    public void updateTask(ScheduledTask task) {
        # 1. 停止旧任务
        stopTask(task.getTaskName());

        # 2. 如果启用，启动新任务
        if (task.getEnabled()) {
            startTask(task);
        }

        # 3. 更新数据库
        taskRepository.save(task);
    }
}
```

```java
// controller/ScheduledTaskController.java
@RestController
@RequestMapping("/api/scheduled-task")
public class ScheduledTaskController {

    @Autowired
    private DynamicSchedulerConfig schedulerConfig;

    @Autowired
    private ScheduledTaskRepository taskRepository;

    /**
     * 获取所有任务
     */
    @GetMapping
    public Result<List<ScheduledTask>> getAll() {
        return Result.success(taskRepository.findAll());
    }

    /**
     * 更新任务
     */
    @PutMapping("/{id}")
    public Result<?> update(@PathVariable Long id, @RequestBody ScheduledTask task) {
        task.setId(id);
        schedulerConfig.updateTask(task);
        return Result.success();
    }

    /**
     * 启用/禁用任务
     */
    @PutMapping("/{id}/toggle")
    public Result<?> toggle(@PathVariable Long id) {
        ScheduledTask task = taskRepository.findById(id).orElseThrow();
        task.setEnabled(!task.getEnabled());
        schedulerConfig.updateTask(task);
        return Result.success();
    }

    /**
     * 重新加载所有任务
     */
    @PostMapping("/reload")
    public Result<?> reload() {
        schedulerConfig.reloadTasks();
        return Result.success();
    }
}
```

## 第五步：分布式调度

### 问题：多实例部署时任务重复执行

```
场景：
3个服务器实例部署应用
→ 每个实例都执行定时任务
→ 任务被执行3次！

例子：
超时订单取消任务
→ 实例1：取消订单
→ 实例2：取消订单（已取消，报错）
→ 实例3：取消订单（已取消，报错）

解决：分布式锁，只有一个实例执行
```

### 代码实现：分布式锁

```java
// task/DistributedScheduledTask.java
package com.taoshu.task;

import lombok.extern.slf4j.Slf4j;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.data.redis.core.StringRedisTemplate;
import org.springframework.scheduling.annotation.Scheduled;
import org.springframework.stereotype.Component;
import java.util.concurrent.TimeUnit;

@Slf4j
@Component
public class DistributedScheduledTask {

    @Autowired
    private StringRedisTemplate redisTemplate;

    private static final String LOCK_KEY_PREFIX = "task:lock:";

    /**
     * 分布式定时任务
     */
    @Scheduled(cron = "0 */1 * * * ?")
    public void distributedTask() {
        String lockKey = LOCK_KEY_PREFIX + "distributedTask";
        String lockValue = String.valueOf(System.currentTimeMillis());

        # 1. 尝试获取锁
        Boolean locked = redisTemplate.opsForValue()
            .setIfAbsent(lockKey, lockValue, 10, TimeUnit.SECONDS);

        if (!Boolean.TRUE.equals(locked)) {
            # 获取锁失败，其他实例正在执行
            log.info("任务正在其他实例执行，跳过");
            return;
        }

        try {
            log.info("获取分布式锁成功，开始执行任务");

            # 2. 执行任务
            doTask();

        } catch (Exception e) {
            log.error("任务执行失败", e);

        } finally {
            # 3. 释放锁（只释放自己的锁）
            String currentValue = redisTemplate.opsForValue().get(lockKey);
            if (lockValue.equals(currentValue)) {
                redisTemplate.delete(lockKey);
                log.info("释放分布式锁");
            }
        }
    }

    private void doTask() {
        # 任务逻辑...
    }
}
```

## ⚠️ 定时任务最佳实践

### 1. 异常处理

```java
@Scheduled(cron = "0 0 1 * * ?")
public void dailyTask() {
    try {
        # 任务逻辑
    } catch (Exception e) {
        log.error("定时任务执行失败", e);
        # 发送告警
    }
}
```

### 2. 超时控制

```java
@Scheduled(cron = "0 0 1 * * ?")
public void taskWithTimeout() {
    try {
        # 使用CompletableFuture实现超时
        CompletableFuture.runAsync(() -> {
            # 任务逻辑
        }).get(5, TimeUnit.MINUTES);  # 最多5分钟

    } catch (TimeoutException e) {
        log.error("任务执行超时");
    } catch (Exception e) {
        log.error("任务执行失败", e);
    }
}
```

## 📊 Django vs Spring Boot 对比

| 功能 | Django | Spring Boot | 难度 |
|------|--------|-------------|------|
| 定时任务 | `django-crontab` / Celery Beat | `@Scheduled` | Spring Boot更简单 |
| Cron表达式 | 同Linux | 同Linux | 相当 |
| 动态配置 | django-crontab-dynamic | TaskScheduler | 相当 |
| 分布式 | Celery | Shedlock | 相当 |
| 监控 | Flower | Actuator | 相当 |

**Django优势**：
- Celery Beat功能强大

**Spring Boot优势**：
- `@Scheduled` 注解简单
- Shedlock轻量级
- 原生支持

| 知识点 | 为什么需要 | 怎么做 |

- **@Scheduled定时**，**Cron表达灵**
