## 🤔 为什么需要搜索功能？

### 真实场景：没有搜索的痛苦

想象一下，你走进一家拥有10万本书的图书馆：

```
❌ 没有搜索的体验：
```

**这就是没有搜索功能的下场**：
- 用户找不到想要的商品 → 转身离开


**搜索功能的价值**：

```java
        return cb.or(

            cb.like(root.get("author"), pattern),
            cb.like(root.get("isbn"), pattern)

    });
```

#### 3. 在Service中实现搜索逻辑

```java
// service/BookService.java
package com.taoshu.service;

import com.taoshu.dto.BookSearchRequest;
import com.taoshu.entity.Book;
import com.taoshu.repository.BookRepository;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.data.domain.Page;
import org.springframework.data.domain.PageRequest;
import org.springframework.data.domain.Pageable;
import org.springframework.data.domain.Sort;
import org.springframework.stereotype.Service;
import org.springframework.util.StringUtils;

@Service
public class BookService {

    @Autowired
    private BookRepository bookRepository;

    # 搜索书籍
    public Page<Book> searchBooks(BookSearchRequest request, int page, int size) {
        # 1. 构建分页参数
        Sort sort = buildSort(request.getSortBy(), request.getSortOrder());
        Pageable pageable = PageRequest.of(page, size, sort);

        # 2. 如果有关键词，执行搜索
        if (StringUtils.hasText(request.getKeyword())) {
            return bookRepository.searchBooks(request.getKeyword(), pageable);
        }

        # 3. 没有关键词，返回所有在售书籍
        return bookRepository.findByIsDeletedFalseAndStatus(1, pageable);
    }

    # 构建排序
    private Sort buildSort(String sortBy, String sortOrder) {
        # 默认按创建时间降序
        if (!StringUtils.hasText(sortBy)) {
            return Sort.by(Sort.Direction.DESC, "createdAt");
        }

        # 排序方向
        Sort.Direction direction = "asc".equalsIgnoreCase(sortOrder)
            ? Sort.Direction.ASC
            : Sort.Direction.DESC;

        # 排序字段白名单（防止SQL注入）
        if (!isValidSortField(sortBy)) {
            sortBy = "createdAt";  # 非法字段，回退到默认
        }

        return Sort.by(direction, sortBy);
    }

    # 验证排序字段是否合法
    private boolean isValidSortField(String field) {
        return field.equals("price") || field.equals("soldQuantity")
            || field.equals("createdAt") || field.equals("viewCount");
    }
}
```

**🔒 安全提示：排序字段白名单**

```java
// ❌ 危险：直接使用用户输入
Sort.by(direction, request.getSortBy())
// 如果用户传入 sortBy = "password; DROP TABLE book" → SQL注入！

// ✅ 安全：白名单验证
private boolean isValidSortField(String field) {
    return field.equals("price") || field.equals("soldQuantity") || ...;
}
```

## 第二步：多条件筛选

### 问题：如何动态组合多个筛选条件？

#### 场景分析

用户可能的筛选组合：

```
场景1：只按分类筛选

场景2：按分类+价格筛选

场景3：按分类+价格+成色筛选
```

```java
// ❌ 糟糕的做法：每个组合写一个方法
```

```java

// 它允许你动态构建WHERE条件，类似Django的Q对象
```

### 代码实现：动态筛选

#### 1. 创建Specification工具类

```java
// spec/BookSpecs.java
package com.taoshu.spec;

import com.taoshu.dto.BookSearchRequest;
import com.taoshu.entity.Book;
import org.springframework.data.jpa.domain.Specification;
import jakarta.persistence.criteria.Predicate;
import java.util.ArrayList;
import java.util.List;

public class BookSpecs {

    # 动态构建查询条件
    public static Specification<Book> buildSpec(BookSearchRequest request) {
        return (root, query, cb) -> {
            List<Predicate> predicates = new ArrayList<>();

            # 1. 基础条件：未删除 + 在售
            predicates.add(cb.equal(root.get("isDeleted"), false));
            if (request.getStatus() != null) {
                predicates.add(cb.equal(root.get("status"), request.getStatus()));
            } else {
                # 默认只显示在售
                predicates.add(cb.equal(root.get("status"), 1));
            }

            # 2. 关键词搜索
            if (request.getKeyword() != null && !request.getKeyword().trim().isEmpty()) {
                String pattern = "%" + request.getKeyword() + "%";
                Predicate titleMatch = cb.like(root.get("title"), pattern);
                Predicate authorMatch = cb.like(root.get("author"), pattern);
                Predicate isbnMatch = cb.like(root.get("isbn"), pattern);
                predicates.add(cb.or(titleMatch, authorMatch, isbnMatch));
            }

            # 3. 分类筛选
            if (request.getCategoryId() != null) {
                predicates.add(cb.equal(root.get("categoryId"), request.getCategoryId()));
            }

            # 4. 价格区间筛选
            if (request.getMinPrice() != null) {
                predicates.add(cb.greaterThanOrEqualTo(root.get("price"), request.getMinPrice()));
            }
            if (request.getMaxPrice() != null) {
                predicates.add(cb.lessThanOrEqualTo(root.get("price"), request.getMaxPrice()));
            }

            # 5. 成色筛选
            if (request.getCondition() != null) {
                predicates.add(cb.equal(root.get("bookCondition"), request.getCondition()));
            }

            # 6. 组合所有条件（AND关系）
            return cb.and(predicates.toArray(new Predicate[0]));
        };
    }
}
```

### 代码实现：搜索历史

#### 1. 设计存储方案

```
选项1：存数据库
user_search_history 表
- 优点：永久保存
- 缺点：每次搜索都写数据库，性能差

选项2：存Redis（推荐！）
key: user:search:{userId}
value: ["Java", "Python", "算法"]
- 优点：读写快
- 缺点：重启后丢失（但历史数据可以容忍丢失）

选项3：混合方案
- Redis存最近7天的历史（快速访问）
- 数据库存所有历史（数据分析）
```

#### 2. Redis存储搜索历史

```java
// service/SearchHistoryService.java
package com.taoshu.service;

import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.data.redis.core.StringRedisTemplate;
import org.springframework.stereotype.Service;
import java.util.List;
import java.util.concurrent.TimeUnit;

@Service
public class SearchHistoryService {

    @Autowired
    private StringRedisTemplate redisTemplate;

    private static final String HISTORY_KEY_PREFIX = "user:search:";
    private static final int MAX_HISTORY_SIZE = 10;  # 最多保存10条

    # 添加搜索历史
    public void addHistory(Long userId, String keyword) {
        String key = HISTORY_KEY_PREFIX + userId;

        # 1. 删除旧的关键词（避免重复）
        redisTemplate.opsForZSet().remove(key, keyword);

        # 2. 添加新的关键词（使用时间戳作为分数）
        redisTemplate.opsForZSet().add(key, keyword, System.currentTimeMillis());

        # 3. 只保留最近的10条
        redisTemplate.opsForZSet().removeRange(key, 0, -MAX_HISTORY_SIZE - 1);

        # 4. 设置过期时间（30天）
        redisTemplate.expire(key, 30, TimeUnit.DAYS);
    }

    # 获取搜索历史（倒序，最新的在前）
    public List<String> getHistory(Long userId) {
        String key = HISTORY_KEY_PREFIX + userId;
        # 倒序获取，返回最近10条
        return redisTemplate.opsForZSet()
            .reverseRange(key, 0, MAX_HISTORY_SIZE - 1);
    }

    # 清空搜索历史
    public void clearHistory(Long userId) {
        String key = HISTORY_KEY_PREFIX + userId;
        redisTemplate.delete(key);
    }

    # 删除单条历史
    public void removeHistory(Long userId, String keyword) {
        String key = HISTORY_KEY_PREFIX + userId;
        redisTemplate.opsForZSet().remove(key, keyword);
    }
}
```

**🤔 为什么用ZSet而不是List？**

```
需求：去重 + 保留最新顺序

方案1：List
LPUSH user:search:1 "Java"
LPUSH user:search:1 "Python"
LPUSH user:search:1 "Java"  # 重复了！
结果：["Java", "Python", "Java"]  # 有重复

方案2：ZSet（推荐！）
ZADD user:search:1 1001 "Java"
ZADD user:search:1 1002 "Python"
ZADD user:search:1 1003 "Java"  # 更新分数
ZREVRANGE user:search:1 0 10
结果：["Java", "Python"]  # 自动去重 + 按时间排序
```

### 代码实现：热门搜索

#### 方案选择

```
方案1：实时统计（不准确）
每次搜索，搜索次数+1
问题：单个用户刷搜索，数据失真

方案2：实时统计 + UV去重（较好）
每次搜索，search_count+1，search_uv+1（按用户去重）
问题：数据量太大

方案3：定时统计（推荐！）
每天凌晨统计昨天的搜索词
问题：有延迟

方案4：混合方案（最优！）
- Redis实时统计（快速响应）
- 数据库持久化（数据分析）
- 定时任务合并数据
```

#### 1. Redis实时统计

```java
// service/HotSearchService.java
package com.taoshu.service;

import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.data.redis.core.StringRedisTemplate;
import org.springframework.data.redis.core.ZSetOperations;
import org.springframework.stereotype.Service;
import java.util.List;
import java.util.Map;
import java.util.stream.Collectors;

@Service
public class HotSearchService {

    @Autowired
    private StringRedisTemplate redisTemplate;

    private static final String HOT_SEARCH_KEY = "search:hot";
    private static final int HOT_SEARCH_SIZE = 20;  # 热搜榜前20

    # 增加搜索次数
    public void incrementSearch(String keyword) {
        redisTemplate.opsForZSet().incrementScore(
            HOT_SEARCH_KEY,
            keyword,
            1
        );
    }

    # 获取热门搜索TOP N
    public List<Map<String, Object>> getHotSearch(int limit) {
        # 获取分数最高的N个关键词
        return redisTemplate.opsForZSet()
            .reverseRangeWithScores(HOT_SEARCH_KEY, 0, limit - 1)
            .stream()
            .map(tuple -> {
                Map<String, Object> item = new HashMap<>();
                item.put("keyword", tuple.getValue());
                item.put("count", tuple.getScore().longValue());
                return item;
            })
            .collect(Collectors.toList());
    }

    # 定时重置（每天凌晨）
    @Scheduled(cron = "0 0 0 * * ?")  # 每天00:00执行
    public void resetHotSearch() {

        # redisTemplate.delete(HOT_SEARCH_KEY);  # 清空重新统计
    }
}
```

## ⚠️ 性能优化

### 问题1：LIKE查询性能差

```sql
-- ❌ 慢查询：前缀模糊匹配（无法使用索引）
SELECT * FROM book WHERE title LIKE '%Java%'

-- ✅ 快查询：后缀模糊匹配（可以使用索引）
SELECT * FROM book WHERE title LIKE 'Java%'
```

**优化方案：Elasticsearch全文搜索**

```
MySQL LIKE：适合小数据量（<10万条）
Elasticsearch：适合大数据量（>10万条），支持分词、拼音、同义词
```

### 问题2：热搜刷榜

```
场景：恶意用户用脚本刷"XXX书籍"100万次
结果：XXX书籍霸占热搜第一

解决方案：
1. 限制单个用户的搜索频率
2. UV去重（同一用户多次搜索只计1次）
3. 设置上限（单个关键词最多显示热度值）
```

```java
// UV去重实现
public void incrementSearch(Long userId, String keyword) {
    String userKey = "search:user:" + userId;
    String today = LocalDate.now().toString();

    # 检查今天是否搜过
    Boolean hasSearched = redisTemplate.opsForSet()
        .isMember(userKey, keyword + ":" + today);

    if (!hasSearched) {
        # 首次搜索，热度+1
        redisTemplate.opsForZSet().incrementScore(HOT_SEARCH_KEY, keyword, 1);
        # 记录已搜索
        redisTemplate.opsForSet().add(userKey, keyword + ":" + today);
        # 7天后过期
        redisTemplate.expire(userKey, 7, TimeUnit.DAYS);
    }
}
```

## 📊 Django vs Spring Boot 对比

| 功能 | Django | Spring Boot | 难度 |
|------|--------|-------------|------|
| 模糊查询 | `Q(title__contains='Java')` | `@Query` 或 `Specification` | Django简单 |
| 动态筛选 | `Q对象组合` | `Specification动态构建` | 相当 |
| 分页 | `Paginator` | `Page<T>` | 相当 |
| 排序 | `order_by()` | `Sort.by()` | 相当 |
| 全文搜索 | Django ORM弱 | 可集成Elasticsearch | Spring Boot强 |
| 热搜统计 | 自定义 | `Redis ZSet` | 相当 |

**Django优势**：
- Q对象语法更简洁
- `__contains` 语法直观

**Spring Boot优势**：
- Specification类型安全
- Redis集成更完善
- 企业级搜索方案（Elasticsearch）成熟
