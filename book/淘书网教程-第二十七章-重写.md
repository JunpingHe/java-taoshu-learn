## 🤔 为什么需要Nginx？

### 真实场景：直接暴露应用的隐患

```
某电商平台直接暴露Spring Boot应用：
┌─────────────────────────────────────────────────────────────┐
│  用户请求 → http://taoshu.com:8080                         │
│      ↓                                                      │
│  问题：                                                     │
│  1. 端口暴露：8080端口不美观                              │
│  2. 静态资源：Spring Boot处理，慢                            │
│  3. 单点故障：应用挂了，服务不可用                          │
│  4. HTTPS：需要应用配置SSL，复杂                            │
│  5. 并发有限：单个应用并发能力有限                          │
│  6. 安全风险：直接暴露应用IP                                │
│                                                             │
│  使用Nginx后：                                             │
│  1. 隐藏端口：用户访问80/443端口                           │
│  2. 静态资源：Nginx直接返回，快                            │
│  3. 负载均衡：多个应用实例                                  │
│  4. HTTPS：Nginx配置SSL                                     │
│  5. 安全防护：隐藏后端IP                                   │
│  6. 缓存加速：静态资源缓存                                  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**Nginx的价值**：
1. **反向代理**：隐藏后端细节
2. **负载均衡**：分发请求到多台服务器
3. **静态资源**：高效处理静态文件
4. **SSL终止**：统一配置HTTPS
5. **缓存加速**：减少后端压力
6. **安全防护**：DDoS防护、IP黑白名单

## 第一步：Nginx安装

### 问题：如何安装Nginx？

```bash

  --name nginx \
  -p 80:80 \
  -p 443:443 \
```

## 第二步：反向代理配置

### 问题：如何配置反向代理？

```nginx
# /etc/nginx/conf.d/taoshu.conf

# 上游服务器（后端应用）
upstream taoshu_backend {
    # 负载均衡策略
    least_conn;  # 最少连接

    # 后端服务器列表
    server 172.18.0.2:8080 weight=1 max_fails=3 fail_timeout=30s;
    server 172.18.0.3:8080 weight=1 max_fails=3 fail_timeout=30s;
    server 172.18.0.4:8080 weight=1 max_fails=3 fail_timeout=30s;

    # keepalive连接
    keepalive 32;
}

server {
    listen 80;
    server_name taoshu.com www.taoshu.com;

    # 日志
    access_log /var/log/nginx/taoshu-access.log;
    error_log /var/log/nginx/taoshu-error.log;

    # 根路径
    location / {
        # 反向代理
        proxy_pass http://taoshu_backend;

        # 传递真实IP
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # 超时配置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;

        # 缓冲配置
        proxy_buffering on;
        proxy_buffer_size 4k;
        proxy_buffers 8 4k;
    }

    # API接口
    location /api/ {
        proxy_pass http://taoshu_backend;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    # 静态资源（直接由Nginx处理）
    location /static/ {
        alias /usr/share/nginx/html/static/;
        expires 7d;  # 缓存7天
        add_header Cache-Control "public, immutable";
    }

    # 健康检查
    location /health {
        access_log off;
        return 200 "healthy\n";
        add_header Content-Type text/plain;
    }
}
```

**参数说明**

```
upstream：定义后端服务器组
server：后端服务器地址
weight：权重（分配更多请求）
max_fails：最大失败次数
fail_timeout：失败超时时间
least_conn：最少连接调度
keepalive：保持连接复用

proxy_pass：代理到后端
proxy_set_header：设置请求头
proxy_connect_timeout：连接超时
proxy_send_timeout：发送超时
proxy_read_timeout：读取超时
```

## 第三步：负载均衡

### 问题：如何实现负载均衡？

#### 负载均衡策略

```nginx
# 策略1：轮询（默认）
upstream backend {
    server 192.168.1.2:8080;
    server 192.168.1.3:8080;
    server 192.168.1.4:8080;
}

# 策略2：权重（根据性能分配）
upstream backend {
    server 192.168.1.2:8080 weight=3;  # 性能好的服务器
    server 192.168.1.3:8080 weight=2;
    server 192.168.1.4:8080 weight=1;
}

# 策略3：最少连接
upstream backend {
    least_conn;
    server 192.168.1.2:8080;
    server 192.168.1.3:8080;
}

# 策略4：IP哈希（同一IP到同一服务器）
upstream backend {
    ip_hash;
    server 192.168.1.2:8080;
    server 192.168.1.3:8080;
}

# 策略5：公平（第三方模块）
upstream backend {
    fair;
    server 192.168.1.2:8080;
    server 192.168.1.3:8080;
}
```

#### 健康检查

```nginx
upstream backend {
    server 192.168.1.2:8080 max_fails=3 fail_timeout=30s;
    server 192.168.1.3:8080 max_fails=3 fail_timeout=30s;

    # 健康检查（需要nginx_plus或第三方模块）
    check interval=3000 rise=2 fall=3 timeout=1000;
}

# 被动健康检查
# max_fails：最大失败次数
# fail_timeout：失败超时时间
```

## 第四步：HTTPS配置

### 问题：如何配置HTTPS？

```
HTTP vs HTTPS：
HTTP：
- 明文传输
- 数据可被窃听
- 容易被劫持

HTTPS：
- 加密传输
- 数据安全
- 身份验证

需要：
1. SSL证书
2. 私钥
3. 配置HTTPS
```

### 代码实现：SSL配置

```nginx
server {
    listen 443 ssl http2;
    server_name taoshu.com www.taoshu.com;

    # SSL证书
    ssl_certificate /etc/nginx/ssl/taoshu.com.crt;
    ssl_certificate_key /etc/nginx/ssl/taoshu.com.key;

    # SSL优化
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES128-GCM-SHA256;
    ssl_prefer_server_ciphers on;
    ssl_session_cache shared:SSL:10m;
    ssl_session_timeout 10m;

    # HSTS（强制HTTPS）
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;

    # 其他配置
    # ...
}

# HTTP → HTTPS重定向
server {
    listen 80;
    server_name taoshu.com www.taoshu.com;

    # 强制HTTPS
    return 301 https://$server_name$request_uri;
}
```

**获取SSL证书**

```
方式1：购买证书
- 代理商：DigiCert, GlobalSign
- 费用：几百到几千元/年

方式2：Let's Encrypt（免费）
- 有效期：90天
- 自动续期
- 推荐：certbot工具

方式3：自签名（仅开发环境）
- openssl生成
- 浏览器会警告
```

## 第五步：静态资源缓存

### 问题：如何加速静态资源？

```nginx
# 静态资源配置
location /static/ {
    alias /usr/share/nginx/html/static/;

    # 缓存配置
    expires 7d;  # 浏览器缓存7天
    add_header Cache-Control "public, immutable";

    # Gzip压缩
    gzip_static on;

    # CORS（跨域）
    add_header Access-Control-Allow-Origin *;
}

location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2)$ {
    expires 30d;
    add_header Cache-Control "public, immutable";
}

# 禁止访问隐藏文件
location ~ /\. {
    deny all;
}
```

## 第六步：Gzip压缩

### 问题：如何减少传输数据量？

```nginx
# Gzip压缩配置
gzip on;
gzip_vary on;
gzip_proxied any;
gzip_comp_level 6;  # 压缩级别（1-9）
gzip_types text/plain text/css text/xml text/javascript
           application/json application/javascript application/xml+rss
           application/rss+xml font/truetype font/opentype
           application/vnd.ms-fontobject image/svg+xml;
gzip_disable "msie6";

# 压缩效果：
# HTML: 70%压缩率
# CSS: 80%压缩率
# JS: 70%压缩率
# 图片：已经压缩，效果不明显
```

## ⚠️ Nginx优化清单

### 性能优化

```
□ Worker进程数：auto
□ Worker连接数：1024
□ Keepalive：超时时间65s
□ Gzip压缩：开启
□ 缓存：静态资源缓存
□ 日志：关闭access_log（生产）
□ 文件描述符：worker_rlimit_nofile
```

### 安全配置

```
□ 隐藏版本号：server_tokens off
□ 限制请求大小：client_max_body_size
□ 限制请求速率：limit_req
□ IP黑白名单：allow/deny
□ 禁止访问隐藏文件
□ SSL配置：TLS 1.2+
□ 安全头：CSP/X-Frame-Options
```
