## 第一步：订单状态设计

### 状态枚举定义

```java
// enums/OrderStatus.java
package com.taoshu.enums;

import lombok.Getter;

@Getter
public enum OrderStatus {

    PENDING_PAYMENT(0, "待支付"),
    PAID(1, "已支付"),
    SHIPPED(2, "已发货"),
    COMPLETED(3, "已完成"),
    CANCELLED(4, "已取消"),
    REFUNDING(5, "退款中"),
    REFUNDED(6, "已退款");

    private final Integer code;
    private final String description;

    OrderStatus(Integer code, String description) {
        this.code = code;
        this.description = description;
    }

    # 根据code获取枚举
    public static OrderStatus fromCode(Integer code) {
        for (OrderStatus status : values()) {
            if (status.getCode().equals(code)) {
                return status;
            }
        }
        throw new IllegalArgumentException("无效的订单状态：" + code);
    }

    # 是否可以取消
    public boolean canCancel() {
        return this == PENDING_PAYMENT;
    }

    # 是否可以支付
    public boolean canPay() {
        return this == PENDING_PAYMENT;
    }

    # 是否可以发货
    public boolean canShip() {
        return this == PAID;
    }

    # 是否可以确认收货
    public boolean canComplete() {
        return this == SHIPPED;
    }

    # 是否可以退款
    public boolean canRefund() {
        return this == PAID || this == SHIPPED;
    }

    # 是否是最终状态
    public boolean isFinal() {
        return this == COMPLETED || this == CANCELLED || this == REFUNDED;
    }
}
```

### 状态流转图

```
                    ┌─────────────────┐
                    │   待支付 (0)    │
                    │                 │
                    │  - 创建订单     │
                    │  - 等待付款     │
                    └────────┬────────┘
                             │
              ┌──────────────┼──────────────┐
              │ 支付/超时      │ 用户取消      │
              ↓               ↓              ↓
       ┌─────────────┐  ┌──────────┐  ┌──────────┐
       │   已支付(1) │  │ 已取消(4)│  │          │
       └──────┬──────┘  └──────────┘  │          │
              │ 商家发货                │          │
              ↓                         │          │
       ┌─────────────┐                  │          │
       │   已发货(2)  │                  │          │
       │  - 物流更新  │                  │          │
       └──────┬──────┘                  │          │
              │ 用户确认/15天自动        │          │
              ↓                         │          │
       ┌─────────────┐                  │          │
       │   已完成(3)  │ ◄────────────────┘          │
       │  - 评价      │                               │
       └─────────────┘                               │
                                                     │
        支付/发货 → 退款                              │
              ↓                                       │
       ┌─────────────┐                               │
       │   退款中(5)  │                               │
       └──────┬──────┘                               │
              │ 退款成功                              │
              ↓                                       │
       ┌─────────────┐                               │
       │   已退款(6)  │ ◄─────────────────────────────┘
       └─────────────┘

流转规则：
- 单向流转（不可逆）
- 待支付 → 已支付 / 已取消
- 已支付 → 已发货 / 退款中
- 已发货 → 已完成 / 退款中
- 退款中 → 已退款
- 已完成/已取消/已退款 = 最终状态
```

## 第二步：状态变更服务

### 代码实现：状态变更

```java
// service/OrderStatusService.java
package com.taoshu.service;

import com.taoshu.entity.Order;
import com.taoshu.enums.OrderStatus;
import com.taoshu.exception.BusinessException;
import com.taoshu.repository.OrderRepository;
import lombok.extern.slf4j.Slf4j;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;
import java.time.LocalDateTime;

@Service
@Slf4j
public class OrderStatusService {

    @Autowired
    private OrderRepository orderRepository;

    @Autowired
    private OrderLogService orderLogService;

    /**
     * 变更订单状态
     * @param orderId 订单ID
     * @param targetStatus 目标状态
     * @param operator 操作人
     * @param remark 备注
     */
    @Transactional(rollbackFor = Exception.class)
    public void changeStatus(Long orderId, OrderStatus targetStatus, String operator, String remark) {
        # 1. 查询订单
        Order order = orderRepository.findById(orderId)
            .orElseThrow(() -> new BusinessException("订单不存在"));

        # 2. 获取当前状态
        OrderStatus currentStatus = OrderStatus.fromCode(order.getStatus());

        # 3. 检查状态流转是否合法
        if (!isValidTransition(currentStatus, targetStatus)) {
            throw new BusinessException(
                String.format("订单状态不能从%s变更为%s",
                    currentStatus.getDescription(),
                    targetStatus.getDescription())
            );
        }

        # 4. 执行状态变更
        Integer oldStatus = order.getStatus();
        order.setStatus(targetStatus.getCode());
        orderRepository.save(order);

        # 5. 记录状态变更日志
        orderLogService.recordChange(
            orderId,
            oldStatus,
            targetStatus.getCode(),
            operator,
            remark
        );

        # 6. 执行状态变更后的业务逻辑
        handleStatusChange(order, currentStatus, targetStatus);

        log.info("订单{}状态变更为：{}", orderId, targetStatus.getDescription());
    }

    /**
     * 检查状态流转是否合法
     */
    private boolean isValidTransition(OrderStatus current, OrderStatus target) {
        # 待支付 → 已支付 / 已取消
        if (current == OrderStatus.PENDING_PAYMENT) {
            return target == OrderStatus.PAID || target == OrderStatus.CANCELLED;
        }

        # 已支付 → 已发货 / 退款中
        if (current == OrderStatus.PAID) {
            return target == OrderStatus.SHIPPED || target == OrderStatus.REFUNDING;
        }

        # 已发货 → 已完成 / 退款中
        if (current == OrderStatus.SHIPPED) {
            return target == OrderStatus.COMPLETED || target == OrderStatus.REFUNDING;
        }

        # 退款中 → 已退款
        if (current == OrderStatus.REFUNDING) {
            return target == OrderStatus.REFUNDED;
        }

        # 最终状态不能再变更
        if (current.isFinal()) {
            return false;
        }

        return false;
    }

    /**
     * 处理状态变更后的业务逻辑
     */
    private void handleStatusChange(Order order, OrderStatus oldStatus, OrderStatus newStatus) {
        switch (newStatus) {
            case PAID:
                # 支付成功后的处理
                onPaid(order);
                break;

            case SHIPPED:
                # 发货后的处理
                onShipped(order);
                break;

            case COMPLETED:
                # 完成后的处理
                onCompleted(order);
                break;

            case CANCELLED:
                # 取消后的处理
                onCancelled(order);
                break;

            case REFUNDED:
                # 退款后的处理
                onRefunded(order);
                break;
        }
    }

    # 支付成功
    private void onPaid(Order order) {
        # 1. 通知商家发货
        notifyMerchant(order);

        # 2. 发送短信给用户
        sendSms(order.getUserId(), "订单支付成功");
    }

    # 发货
    private void onShipped(Order order) {
        # 1. 发送物流通知
        sendSms(order.getUserId(), "订单已发货");
    }

    # 完成
    private void onCompleted(Order order) {
        # 1. 给商家打款
        settleMerchant(order);

        # 2. 邀请用户评价
        inviteReview(order);
    }

    # 取消
    private void onCancelled(Order order) {
        # 1. 恢复库存
        restoreStock(order);

        # 2. 如果已支付，发起退款
        if (order.getPaymentTime() != null) {
            initiateRefund(order);
        }
    }

    # 退款
    private void onRefunded(Order order) {
        # 1. 恢复库存
        restoreStock(order);

        # 2. 通知用户
        sendSms(order.getUserId(), "退款成功");
    }
```

## 第三步：超时自动取消

### 问题：用户下单后一直不付款

```
场景：
用户A下单《Java核心技术》，库存扣减1本
      → 用户A不付款，也不取消
      → 库存一直被占用
      → 其他用户买不到
      → 15天后用户B买不到，流失

损失：
- 库存浪费
- 销售机会损失
- 用户体验差

解决：
超时自动取消（30分钟内不付款，自动取消）
```

### 代码实现：定时取消

```java
// task/OrderTimeoutTask.java
package com.taoshu.task;

import com.taoshu.entity.Order;
import com.taoshu.enums.OrderStatus;
import com.taoshu.repository.OrderRepository;
import com.taoshu.service.OrderStatusService;
import lombok.extern.slf4j.Slf4j;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.scheduling.annotation.Scheduled;
import org.springframework.stereotype.Component;
import java.time.LocalDateTime;
import java.util.List;

@Component
@Slf4j
public class OrderTimeoutTask {

    @Autowired
    private OrderRepository orderRepository;

    @Autowired
    private OrderStatusService orderStatusService;

    /**
     * 取消超时未支付订单
     * 每1分钟执行一次
     */
    @Scheduled(fixedRate = 60000)
    public void cancelTimeoutOrders() {
        # 查询30分钟前创建的待支付订单
        LocalDateTime timeout = LocalDateTime.now().minusMinutes(30);
        List<Order> timeoutOrders = orderRepository
            .findByStatusAndCreatedAtBefore(OrderStatus.PENDING_PAYMENT.getCode(), timeout);

        log.info("发现{}个超时订单", timeoutOrders.size());

        for (Order order : timeoutOrders) {
            try {
                # 变更状态为已取消
                orderStatusService.changeStatus(
                    order.getId(),
                    OrderStatus.CANCELLED,
                    "SYSTEM",
                    "超时未支付自动取消"
                );

                log.info("订单{}超时取消", order.getOrderNo());

            } catch (Exception e) {
                log.error("取消订单{}失败", order.getId(), e);
            }
        }
    }
}
```

**优化：延迟队列**

```java
// 使用延迟队列，减少定时任务查询
// service/OrderDelayQueue.java
@Component
@Slf4j
public class OrderDelayQueue {

    @Autowired
    private RabbitTemplate rabbitTemplate;

    @Autowired
    private OrderStatusService orderStatusService;

    private static final String DELAY_EXCHANGE = "order.delay.exchange";
    private static final String DELAY_QUEUE = "order.delay.queue";

    /**
     * 发送延迟消息（订单创建时调用）
     */
    public void sendDelayMessage(Long orderId, Integer timeoutMinutes) {
        OrderDelayMessage message = new OrderDelayMessage();
        message.setOrderId(orderId);
        message.setTimeoutMinutes(timeoutMinutes);

        rabbitTemplate.convertAndSend(
            DELAY_EXCHANGE,
            DELAY_QUEUE,
            message,
            msg -> {
                # 设置延迟时间（毫秒）
                msg.getMessageProperties().setDelay(timeoutMinutes * 60 * 1000);
                return msg;
            }
        );
    }

    /**
     * 消费延迟消息
     */
    @RabbitListener(queues = DELAY_QUEUE)
    public void handleDelayMessage(OrderDelayMessage message) {
        try {
            Order order = orderRepository.findById(message.getOrderId())
                .orElse(null);

            # 订单仍然是待支付状态，取消订单
            if (order != null && order.getStatus() == OrderStatus.PENDING_PAYMENT.getCode()) {
                orderStatusService.changeStatus(
                    order.getId(),
                    OrderStatus.CANCELLED,
                    "SYSTEM",
                    "超时未支付自动取消"
                );

                log.info("订单{}超时取消（延迟队列）", order.getOrderNo());
            }

        } catch (Exception e) {
            log.error("处理延迟消息失败", e);
        }
    }
}
```

**对比：定时任务 vs 延迟队列**

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| 定时任务 | 简单，不需要MQ | 每次全量查询，浪费资源 | 小规模 |
| 延迟队列 | 精确，性能好 | 需要MQ，复杂 | 大规模 |

## 第五步：自动确认收货

### 问题：用户一直不确认收货

```
场景：
商家发货 → 用户收到货
      → 用户忘记确认收货
      → 商家的钱一直被冻结
      → 商家资金周转困难

解决：
发货后15天，自动确认收货
```

### 代码实现：自动确认

```java
// task/OrderAutoConfirmTask.java
@Component
@Slf4j
public class OrderAutoConfirmTask {

    @Autowired
    private OrderRepository orderRepository;

    @Autowired
    private OrderStatusService orderStatusService;

    /**
     * 自动确认收货
     * 每天凌晨1点执行
     */
    @Scheduled(cron = "0 0 1 * * ?")
    public void autoConfirmOrders() {
        # 查询15天前发货的订单
        LocalDateTime deadline = LocalDateTime.now().minusDays(15);
        List<Order> orders = orderRepository
            .findShippedOrdersBefore(deadline);

        log.info("发现{}个需要自动确认的订单", orders.size());

        for (Order order : orders) {
            try {
                orderStatusService.changeStatus(
                    order.getId(),
                    OrderStatus.COMPLETED,
                    "SYSTEM",
                    "发货超过15天，自动确认收货"
                );

                log.info("订单{}自动确认收货", order.getOrderNo());

            } catch (Exception e) {
                log.error("自动确认订单{}失败", order.getId(), e);
            }
        }
    }
}
```

### 代码实现：统计服务

```java
// service/OrderStatisticsService.java
package com.taoshu.service;

import com.taoshu.repository.OrderRepository;
import lombok.Data;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;
import java.time.LocalDate;
import java.time.LocalDateTime;
import java.util.List;
import java.util.Map;
import java.util.stream.Collectors;

@Service
public class OrderStatisticsService {

    @Autowired
    private OrderRepository orderRepository;

    /**
     * 订单状态统计
     */
    public OrderStatisticsDTO getStatistics(LocalDate startDate, LocalDate endDate) {
        LocalDateTime start = startDate.atStartOfDay();
        LocalDateTime end = endDate.plusDays(1).atStartOfDay();

        List<Order> orders = orderRepository.findByCreatedAtBetween(start, end);

        OrderStatisticsDTO statistics = new OrderStatisticsDTO();
        statistics.setStartDate(startDate);
        statistics.setEndDate(endDate);

        # 按状态分组统计
        Map<Integer, Long> statusCount = orders.stream()
            .collect(Collectors.groupingBy(Order::getStatus, Collectors.counting()));

        statistics.setTotalCount(orders.size());
        statistics.setPendingPaymentCount(statusCount.getOrDefault(0, 0L));
        statistics.setPaidCount(statusCount.getOrDefault(1, 0L));
        statistics.setShippedCount(statusCount.getOrDefault(2, 0L));
        statistics.setCompletedCount(statusCount.getOrDefault(3, 0L));
        statistics.setCancelledCount(statusCount.getOrDefault(4, 0L));

        # 金额统计
        statistics.setTotalAmount(
            orders.stream()
                .filter(o -> o.getStatus() >= 1)  # 已支付及以后
                .map(Order::getPayAmount)
                .reduce(java.math.BigDecimal.ZERO, java.math.BigDecimal::add)
        );

        return statistics;
    }

    /**
     * 热销商品统计
     */
    public List<BookSalesDTO> getTopSellingBooks(LocalDate startDate, LocalDate endDate, int limit) {
        return orderRepository.findTopSellingBooks(
            startDate.atStartOfDay(),
            endDate.plusDays(1).atStartOfDay(),
            PageRequest.of(0, limit)
        );
    }

    @Data
    public static class OrderStatisticsDTO {
        private LocalDate startDate;
        private LocalDate endDate;
        private Long totalCount;
        private Long pendingPaymentCount;
        private Long paidCount;
        private Long shippedCount;
        private Long completedCount;
        private Long cancelledCount;
        private java.math.BigDecimal totalAmount;
    }

    @Data
    public static class BookSalesDTO {
        private Long bookId;
        private String bookTitle;
        private Long salesCount;
        private java.math.BigDecimal salesAmount;
    }
}
```

```java
@Component
public class OrderStatusPublisher {

    @Autowired
    private ApplicationEventPublisher eventPublisher;

    public void publishStatusChanged(Order order, OrderStatus oldStatus, OrderStatus newStatus) {
        OrderStatusChangedEvent event = new OrderStatusChangedEvent(
            this,
            order.getId(),
            oldStatus,
            newStatus

        eventPublisher.publishEvent(event);

            notifyMerchant(event.getOrderId());
        }
    }
}
```

### 3. 状态机模式

```java
# 使用状态机模式，更规范
# 使用Spring State Machine项目
@Configuration
@EnableStateMachine
public class OrderStateMachineConfig extends StateMachineConfigurerAdapter<OrderStatus, OrderEvent> {

    @Override
    public void configure(StateMachineStateConfigurer<OrderStatus, OrderEvent> states) throws Exception {
        states
            .withStates()
            .initial(OrderStatus.PENDING_PAYMENT)
            .states(EnumSet.allOf(OrderStatus.class));
    }

    @Override
    public void configure(StateMachineTransitionConfigurer<OrderStatus, OrderEvent> transitions) throws Exception {
        transitions
            .withExternal()
                .source(OrderStatus.PENDING_PAYMENT)
                .target(OrderStatus.PAID)
                .event(OrderEvent.PAY)
            .and()
            .withExternal()
                .source(OrderStatus.PAID)
                .target(OrderStatus.SHIPPED)
                .event(OrderEvent.SHIP);
    }
}
```

## 📊 Django vs Spring Boot 对比

| 功能 | Django | Spring Boot | 难度 |
|------|--------|-------------|------|
| 状态枚举 | `choices` | `enum` | 相当 |
| 状态检查 | `if status == x` | `canXxx()` 方法 | Spring Boot更规范 |
| 定时任务 | `django-crontab` | `@Scheduled` | Spring Boot更简单 |
| 状态历史 | 手动实现 | 同样手动实现 | 相当 |
| 事件驱动 | 信号机制 | `@EventListener` | 相当 |

**Django优势**：
- 信号机制简单

**Spring Boot优势**：
- `@Scheduled` 注解简单
- `@EventListener` 类型安全
- State Machine支持
