## 第一步：SQL注入防护

### 问题：什么是SQL注入？

```
场景：用户登录
输入：用户名=admin，密码=123456

SQL拼接（不安全）：
String sql = "SELECT * FROM user WHERE username='" + username + "' AND password='" + password + "'";
# 实际执行：
# SELECT * FROM user WHERE username='admin' AND password='123456'

黑客输入：
username = admin' OR '1'='1
password = anything

实际执行：
SELECT * FROM user WHERE username='admin' OR '1'='1' AND password='anything'
# 结果：'1'='1'永远为真，绕过密码验证！

后果：
- 任意账户登录
- 窃取所有数据
- 删除数据库
```

### 防御方案：参数化查询

```java
// ❌ 不安全：SQL拼接
@Query("SELECT u FROM User u WHERE u.username='" + username + "'")
User findByUsernameUnsafe(String username);

// ✅ 安全：参数化查询
@Query("SELECT u FROM User u WHERE u.username = :username")
User findByUsername(@Param("username") String username);

// ✅ 安全：Repository方法
User findByUsername(String username);

// ✅ 安全：JDBC PreparedStatement
String sql = "SELECT * FROM user WHERE username = ? AND password = ?";
PreparedStatement ps = connection.prepareStatement(sql);
ps.setString(1, username);
ps.setString(2, password);
```

**参数化查询原理**

```
参数化查询：
SELECT * FROM user WHERE username = ? AND password = ?

数据库处理：
1. 预编译SQL（生成执行计划）
2. 参数作为值传入
3. 参数不会被当作SQL执行

即使用户输入 admin' OR '1'='1：
1. 预编译：SELECT * FROM user WHERE username = ? AND password = ?
2. 参数1：admin' OR '1'='1（当作字符串值）
3. 结果：查找用户名为 "admin' OR '1'='1" 的用户（不存在）
```

## 第二步：XSS攻击防护

### 问题：什么是XSS攻击？

```
场景：用户评论
输入：<script>alert(document.cookie)</script>

不安全处理：
1. 直接存储到数据库
2. 返回前端
3. 浏览器执行<script>标签
4. Cookie被窃取

XSS类型：
1. 反射型：URL参数中的恶意脚本被反射执行
2. 存储型：恶意脚本存储到数据库，每次访问都执行
3. DOM型：修改页面DOM结构

后果：
- Cookie被窃取
- 会话劫持
- 恶意操作
- 钓鱼攻击
```

### 防御方案

#### 1. 输入过滤

```java
// util/XssUtil.java
package com.taoshu.util;

import org.springframework.web.util.HtmlUtils;
import org.jsoup.Jsoup;
import org.jsoup.safety.Safelist;

public class XssUtil {

    /**
     * 清理HTML，防止XSS
     */
    public static String clean(String html) {
        if (html == null) {
            return null;
        }

        # 方法1：移除所有标签
        // return HtmlUtils.htmlEscape(html);

        # 方法2：白名单保留安全标签（推荐）
        return Jsoup.clean(html, Safelist.basic());
        # 允许的标签：b, em, i, strong, u, p, br
        # 移除的标签：script, iframe, object, embed等危险标签
    }

    /**
     * 严格的文本清理（不允许任何HTML）
     */
    public static String cleanText(String text) {
        if (text == null) {
            return null;
        }
        return HtmlUtils.htmlEscape(text);
    }
}
```

#### 2. 输出编码

```java
// 在模板引擎中编码
# Thymeleaf默认转义，安全
<p th:text="${comment.content}">评论内容</p>

# 禁止使用th:utext（不转义）
# <p th:utext="${comment.content}">评论内容</p>

# 前端也需要编码
function escapeHtml(text) {
    const map = {
        '&': '&amp;',
        '<': '&lt;',
        '>': '&gt;',
        '"': '&quot;',
        "'": '&#039;'
    };
    return text.replace(/[&<>"']/g, function(m) { return map[m]; });
}
```

#### 3. Content-Security-Policy

```yaml
# application.yml
spring:
  security:
    headers:
      content-security-policy: "default-src 'self'; script-src 'self' 'unsafe-inline' 'unsafe-eval'; style-src 'self' 'unsafe-inline';"
```

## 第三步：CSRF攻击防护

### 问题：什么是CSRF攻击？

```
场景：用户已登录淘书网
1. 用户访问恶意网站（钓鱼邮件）
2. 恶意网站有一个表单：
   <form action="http://taoshu.com/api/order/create" method="POST">
     <input name="bookId" value="123">
     <input type="submit" value="领取优惠券">
   </form>
3. 用户点击按钮
4. 浏览器携带Cookie发送请求
5. 用户在不知情下创建了订单

CSRF条件：
1. 用户已登录目标网站
2. 目标网站没有CSRF防护
3. 用户访问恶意网站
4. 恶意网站伪造请求

防御：
1. CSRF Token
2. SameSite Cookie
3. 验证Referer
```

### 防御方案：CSRF Token

```java
// config/SecurityConfig.java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .csrf(csrf -> csrf
                .csrfTokenRepository(CookieCsrfTokenRepository.withHttpOnlyFalse())
                # 忽略查询接口（只保护修改操作）
                .ignoringRequestMatchers("/api/**")
            )
            # ...

        return http.build();
    }
}
```

**前端处理**

```javascript
// 获取CSRF Token
fetch('/api/csrf-token')
  .then(res => res.json())
  .then(data => {
    const token = data.token;

    // 请求时携带Token
    fetch('/api/order/create', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'X-CSRF-TOKEN': token
      },
      body: JSON.stringify({...})
    });
  });
```

## 第四步：密码安全

### 问题：密码如何安全存储？

```
❌ 明文存储（极不安全）
数据库泄露 → 所有密码直接暴露

❌ 可逆加密（不安全）
加密后可以解密 → 密钥泄露风险

❌ MD5/SHA1（不安全）
彩虹表攻击 → 可以反向破解

✅ BCrypt（推荐）
单向加密 + 自适应 + 盐值
```

### 代码实现：BCrypt加密

```java
// config/SecurityConfig.java
@Bean
public PasswordEncoder passwordEncoder() {
    return new BCryptPasswordEncoder();
}

// service/UserService.java
@Service
public class UserService {

    @Autowired
    private PasswordEncoder passwordEncoder;

    public void register(UserRegisterRequest request) {
        User user = new User();
        user.setPhone(request.getPhone());

        # 加密密码
        String encodedPassword = passwordEncoder.encode(request.getPassword());
        user.setPassword(encodedPassword);

        userRepository.save(user);
    }

    public boolean login(String phone, String rawPassword) {
        User user = userRepository.findByPhone(phone)
            .orElseThrow(() -> new BusinessException("用户不存在"));

        # 验证密码
        return passwordEncoder.matches(rawPassword, user.getPassword());
    }
}
```

**BCrypt特性**

```
1. 单向加密：无法解密
2. 自适应：计算强度可调整
3. 盐值内置：每个密码不同盐值
4. 慢速：故意设计得慢（防止暴力破解）

示例：
密码：123456
加密1：$2a$10$N9qo8uLOickgx2ZMRZoMyeIjZAgcfl7p92ldGxad68LJZdL17lhWy
加密2：$2a$10$g6aVPAvNqJ8p8hLd2vVQOjZ1Z5rY8jZ8nX9bLkMmO0PpQqRrSsT7U
# 同一密码，每次加密结果不同（盐值不同）
```

## 第五步：敏感信息保护

### 问题：如何保护敏感信息？

```
敏感信息：
1. 数据库密码
2. API密钥（支付宝、微信）
3. JWT密钥
4. 第三方凭证

危险操作：
1. 硬编码在代码中
   private static final String DB_PASSWORD = "123456";  # ❌

2. 提交到GitHub
   # 代码公开，密码泄露

3. 使用弱密码
   # 容易被破解
```

### 防御方案：环境变量

```java
// application.yml
spring:
  datasource:
    url: ${DB_URL:jdbc:mysql://localhost:3306/taoshu}
    username: ${DB_USERNAME:root}
    password: ${DB_PASSWORD}  # 从环境变量读取

# JWT密钥
jwt:
  secret: ${JWT_SECRET}  # 从环境变量读取

# 支付宝配置
alipay:
  app-id: ${ALIPAY_APP_ID}
  private-key: ${ALIPAY_PRIVATE_KEY}
  public-key: ${ALIPAY_PUBLIC_KEY}
```

```bash
# 设置环境变量（Linux/Mac）
export DB_PASSWORD='your_strong_password'
export JWT_SECRET='your_jwt_secret_key'
export ALIPAY_PRIVATE_KEY='your_private_key'

# Windows
set DB_PASSWORD=your_strong_password
set JWT_SECRET=your_jwt_secret_key
```

**生产环境最佳实践**

```
1. 使用配置中心（Spring Cloud Config / Apollo）
2. 使用密钥管理服务（AWS KMS / HashiCorp Vault）
3. 定期轮换密钥
4. 最小权限原则
5. 审计日志
```

## 第六步：接口安全

### 问题：如何保护API接口？

```
安全层次：
1. 认证（Authentication）：你是谁
   - JWT Token
   - Session
   - OAuth2

2. 鉴权（Authorization）：你能做什么
   - 角色权限（RBAC）
   - 接口权限
   - 数据权限

3. 审计（Audit）：你做了什么
   - 操作日志
   - 访问日志
```

### 代码实现：JWT认证

```java
// filter/JwtAuthenticationFilter.java
@Component
public class JwtAuthenticationFilter extends OncePerRequestFilter {

    @Autowired
    private JwtUtil jwtUtil;

    @Override
    protected void doFilterInternal(HttpServletRequest request,
                                    HttpServletResponse response,
                                    FilterChain chain) throws ServletException, IOException {
        # 1. 从请求头获取Token
        String token = request.getHeader("Authorization");
        if (token != null && token.startsWith("Bearer ")) {
            token = token.substring(7);

            try {
                # 2. 验证Token
                Claims claims = jwtUtil.parseToken(token);
                Long userId = claims.get("userId", Long.class);

                # 3. 设置到SecurityContext
                UsernamePasswordAuthenticationToken auth =
                    new UsernamePasswordAuthenticationToken(userId, null, Collections.emptyList());
                SecurityContextHolder.getContext().setAuthentication(auth);

            } catch (Exception e) {
                # Token无效
                response.setStatus(HttpServletResponse.SC_UNAUTHORIZED);
                response.getWriter().write("{\"code\":401,\"message\":\"Token无效\"}");
                return;
            }
        }

        chain.doFilter(request, response);
    }
}
```

## 📊 Django vs Spring Boot 对比

| 安全特性 | Django | Spring Boot | 难度 |
|---------|--------|-------------|------|
| SQL注入 | ORM自动防护 | 参数化查询 | 相当 |
| XSS防护 | 默认转义 | 需要配置 | Django更安全 |
| CSRF防护 | 默认启用 | 需要配置 | Django更方便 |
| 密码加密 | make_password | BCrypt | 相当 |
| 认证鉴权 | Django Auth | Spring Security | 相当 |
| 依赖检查 | pip check | OWASP Dependency Check | 相当 |

**Django优势**：
- 默认安全配置好
- CSRF自动防护

**Spring Boot优势**：
- Spring Security更灵活
- 企业级安全方案
