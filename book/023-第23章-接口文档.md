# 第二十三章：接口文档

> **小王的疑问**：前端开发问我订单接口的参数格式，我该怎么告诉他？每次接口变动都要手动更新文档吗？Django REST Framework有Swagger，Spring Boot呢？

---

## 🤔 为什么需要接口文档？

### 真实场景：没有文档的痛苦

```
某电商平台没有完善的接口文档：
┌─────────────────────────────────────────────────────────────┐
│  前端开发："订单创建接口怎么调用？"                          │
│  后端开发："看一下代码吧..."                               │
│  前端开发："这么多代码，看不懂啊..."                        │
│                                                             │
│  前端开发："用户注册需要传什么参数？"                        │
│  后端开发："用户名、密码、手机号..."                         │
│  前端开发："手机号格式有什么要求？"                         │
│  后端开发："我查一下...应该是11位..."                       │
│                                                             │
│  一周后：                                                   │
│  前端开发："接口报错了！"                                   │
│  后端开发："哦，我改了接口，忘记告诉你了..."                │
│                                                             │
│  问题：                                                     │
│  - 沟通成本高                                              │
│  - 接口变更不同步                                          │
│  - 前后端联调慢                                            │
│  - 文档维护困难                                            │
│                                                             │
│  有完善的接口文档：                                         │
│  1. 代码注解 → 自动生成文档                                │
│  2. 在线查看 → Swagger UI                                 │
│  3. 实时测试 → 直接在文档测试接口                          │
│  4. 代码变更 → 文档自动更新                                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**接口文档的价值**：
1. **降低沟通成本**：前端自文档查看
2. **提高开发效率**：快速了解接口
3. **减少联调时间**：在线测试接口
4. **保持文档同步**：自动生成
5. **方便团队协作**：统一规范

## 第一步：Swagger配置

### 问题：Swagger是什么？

```
Swagger是什么？
- API文档生成工具
- 从代码注解自动生成文档
- 提供在线测试界面

Swagger组成：
1. Swagger注解：标注接口信息
2. Swagger Core：解析注解
3. Swagger UI：展示文档 + 测试

Spring Boot集成：
- 传统：springfox-swagger2（已停止维护）
- 推荐：springdoc-openapi（积极维护）
```

### 代码实现：配置

#### 1. 添加依赖

```xml
<!-- pom.xml -->
<dependency>
    <groupId>org.springdoc</groupId>
    <artifactId>springdoc-openapi-starter-webmvc-ui</artifactId>
    <version>2.2.0</version>
</dependency>
```

#### 2. 配置类

```java
// config/OpenApiConfig.java
package com.taoshu.config;

import io.swagger.v3.oas.models.OpenAPI;
import io.swagger.v3.oas.models.info.Contact;
import io.swagger.v3.oas.models.info.Info;
import io.swagger.v3.oas.models.info.License;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class OpenApiConfig {

    @Bean
    public OpenAPI customOpenAPI() {
        return new OpenAPI()
            .info(new Info()
                .title("淘书网 API文档")
                .version("1.0.0")
                .description("淘书网电商平台接口文档")
                .contact(new Contact()
                    .name("淘书网技术团队")
                    .email("tech@taoshu.com"))
                .license(new License()
                    .name("Apache 2.0")
                    .url("https://www.apache.org/licenses/LICENSE-2.0.html"))
            );
    }
}
```

#### 3. 访问文档

```
启动应用后访问：
Swagger UI: http://localhost:8080/swagger-ui.html
OpenAPI JSON: http://localhost:8080/v3/api-docs
```

## 第二步：接口注解

### 问题：如何标注接口信息？

```java
// controller/OrderController.java
package com.taoshu.controller;

import io.swagger.v3.oas.annotations.Operation;
import io.swagger.v3.oas.annotations.Parameter;
import io.swagger.v3.oas.annotations.tags.Tag;
import io.swagger.v3.oas.annotations.enums.ParameterIn;
import jakarta.validation.Valid;
import org.springframework.web.bind.annotation.*;

@RestController
@RequestMapping("/api/order")
@Tag(name = "订单管理", description = "订单相关接口")  # 接口分组
public class OrderController {

    @Operation(
        summary = "创建订单",  # 接口标题
        description = "用户下单创建订单，返回订单信息"  # 详细说明
    )
    @PostMapping
    public Result<Order> createOrder(
            @Parameter(
                description = "订单创建请求",
                required = true,
                example = "{\"userId\":1,\"items\":[{\"bookId\":123,\"quantity\":2}]}"
            )
            @RequestBody @Valid CreateOrderRequest request) {
        # ...
    }

    @Operation(
        summary = "查询订单",
        description = "根据订单ID查询订单详情"
    )
    @GetMapping("/{id}")
    public Result<Order> getOrder(
            @Parameter(
                description = "订单ID",
                required = true,
                example = "1",
                in = ParameterIn.PATH  # 路径参数
            )
            @PathVariable Long id) {
        # ...
    }

    @Operation(
        summary = "取消订单",
        description = "取消待支付的订单"
    )
    @PutMapping("/{id}/cancel")
    public Result<?> cancelOrder(
            @Parameter(description = "订单ID", required = true)
            @PathVariable Long id,

            @Parameter(description = "取消原因", example = "不想要了")
            @RequestParam(required = false) String reason) {
        # ...
    }
}
```

## 第三步：Model注解

### 问题：如何标注实体信息？

```java
// dto/CreateOrderRequest.java
package com.taoshu.dto;

import io.swagger.v3.oas.annotations.media.Schema;
import jakarta.validation.constraints.*;
import lombok.Data;
import java.util.List;

@Data
@Schema(description = "创建订单请求")
public class CreateOrderRequest {

    @Schema(
        description = "用户ID",
        example = "1",
        required = true
    )
    @NotNull(message = "用户ID不能为空")
    private Long userId;

    @Schema(
        description = "订单明细列表",
        required = true
    )
    @NotEmpty(message = "订单明细不能为空")
    private List<OrderItemRequest> items;

    @Schema(
        description = "收货人姓名",
        example = "张三",
        required = true
    )
    @NotBlank(message = "收货人姓名不能为空")
    private String receiverName;

    @Schema(
        description = "联系电话",
        example = "13800138000",
        required = true
    )
    @NotBlank(message = "联系电话不能为空")
    @Pattern(regexp = "^1\\d{10}$", message = "手机号格式错误")
    private String receiverPhone;

    @Schema(
        description = "收货地址",
        example = "北京市朝阳区xxx",
        required = true
    )
    @NotBlank(message = "收货地址不能为空")
    private String receiverAddress;

    @Data
    @Schema(description = "订单明细")
    public static class OrderItemRequest {

        @Schema(description = "商品ID", example = "123", required = true)
        @NotNull(message = "商品ID不能为空")
        private Long bookId;

        @Schema(description = "购买数量", example = "2", required = true)
        @Min(value = 1, message = "数量至少为1")
        private Integer quantity;
    }
}
```

## 第六步：Knife4j增强UI

```
Knife4j是什么？

特性：
1. 离线文档（Markdown）

```

### 代码实现：Knife4j

```xml
<!-- pom.xml -->
<dependency>
    <groupId>com.github.xiaoymin</groupId>
    <artifactId>knife4j-openapi3-jakarta-spring-boot-starter</artifactId>
    <version>4.3.0</version>
```

```yaml
# application.yml
knife4j:
  enable: true
  setting:
    language: zh_cn  # 中文
```

## ⚠️ 文档最佳实践

### 1. 注解规范

```java
# 好的注解
@Operation(
    summary = "创建订单",  # 简短描述
    description = "用户下单创建订单，返回订单信息"  # 详细说明
)
public Result<Order> createOrder() { }

# 不好的注解
@Operation(summary = "接口")  # 太简单
public Result<Order> createOrder() { }
```

### 2. Example示例

```java
@Schema(example = "13800138000")  # 提供示例值
private String phone;

@Parameter(example = "1")  # 参数示例
@PathVariable Long id;
```

### 3. 分组合理

```java
# 按功能模块分组
@Tag(name = "订单管理", description = "订单相关接口")

# 按业务角色分组
@Tag(name = "用户端", description = "用户端接口")
@Tag(name = "管理端", description = "管理端接口")
```

### 4. 生产环境关闭

```java
// config/Knife4jConfig.java
@Profile({"dev", "test"})  # 只在开发/测试环境启用
@Configuration
public class Knife4jConfig {
    # ...
}

// 或者配置文件
knife4j:
  production: true  # 生产环境关闭访问
```

## 📊 Django vs Spring Boot 对比

| 功能 | Django | Spring Boot | 难度 |
|------|--------|-------------|------|
| 文档生成 | DRF Auto Schema | springdoc-openapi | 相当 |
| 注解标注 | @api_view | @Operation | 相当 |
| 在线测试 | DRFBrowsableAPI | Swagger UI | 相当 |
| UI美化 | drf-yasg | Knife4j | 相当 |
| 中文支持 | 中文文档 | Knife4j中文 | 相当 |

**Django优势**：
- DRF集成更紧密

**Spring Boot优势**：
- Knife4j UI更美观
- 注解更直观
- 配置更简单
