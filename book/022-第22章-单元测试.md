## 🤔 为什么需要单元测试？

### 真实场景：没有测试的灾难

```
某电商平台没有完善的测试：
┌─────────────────────────────────────────────────────────────┐
│  开发流程：                                                 │
│  1. 写代码 → 2. 手动测一下 → 3. 上线                        │
│                                                             │
│  上线后：                                                   │
│  - Bug：订单金额计算错误                                    │
│  - Bug：库存扣减成负数                                      │
│  - Bug：用户可以重复下单                                    │
│  - Bug：优惠券重复使用                                      │
│  ...                                                       │
│                                                             │
│  影响：                                                     │
│  - 用户投诉                                                │
│  - 资金损失                                                │
│  - 紧急修复 → 引入新Bug                                    │
│  - 恶性循环                                                │
│                                                             │
│  有完善的测试：                                             │
│  1. 写代码 → 2. 写测试 → 3. 运行测试                       │
│  4. 测试通过 → 5. 代码审查 → 6. 上线                       │
│                                                             │
│  结果：                                                     │
│  - Bug在测试阶段发现                                       │
│  - 上线后Bug减少90%                                        │
│  - 代码重构有信心                                          │
│  - 维护成本降低                                            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**单元测试的价值**：
1. **提前发现Bug**：开发阶段发现，成本最低
2. **重构信心**：有测试保护，重构不担心
3. **代码文档**：测试就是最好的文档
4. **设计指导**：好测试倒逼好设计
5. **提高质量**：减少线上Bug

## 第一步：JUnit 5基础

### 问题：如何写第一个测试？

```java
// service/Calculator.java
package com.taoshu.service;

public class Calculator {
    public int add(int a, int b) {
        return a + b;
    }

    public int divide(int a, int b) {
        return a / b;
    }
}
```

```java
// test/service/CalculatorTest.java
package com.taoshu.service;

import org.junit.jupiter.api.Test;
import org.junit.jupiter.api.DisplayName;
import static org.junit.jupiter.api.Assertions.*;

class CalculatorTest {

    @Test
    @DisplayName("测试加法")
    void testAdd() {
        # given
        Calculator calculator = new Calculator();
        int a = 2;
        int b = 3;

        # when
        int result = calculator.add(a, b);

        # then
        assertEquals(5, result, "2 + 3 应该等于 5");
    }

    @Test
    @DisplayName("测试除法")
    void testDivide() {
        Calculator calculator = new Calculator();

        int result = calculator.divide(6, 2);

        assertEquals(3, result);
    }

    @Test
    @DisplayName("测试除以零")
    void testDivideByZero() {
        Calculator calculator = new Calculator();

        # 断言抛出异常
        assertThrows(ArithmeticException.class, () -> {
            calculator.divide(1, 0);
        });
    }
}
```

**JUnit 5常用注解**

| 注解 | 说明 |
|------|------|
| `@Test` | 测试方法 |
| `@DisplayName` | 测试显示名称 |
| `@BeforeEach` | 每个测试前执行 |
| `@AfterEach` | 每个测试后执行 |
| `@BeforeAll` | 所有测试前执行（静态） |
| `@AfterAll` | 所有测试后执行（静态） |
| `@Disabled` | 禁用测试 |
| `@RepeatedTest` | 重复执行 |

## 第二步：断言

### 问题：如何验证结果？

```java
// 常用断言
@Test
void testAssertions() {
    # 1. 相等断言
    assertEquals(4, 2 + 2);
    assertEquals("hello", "HELLO".toLowerCase());

    # 2. 不等断言
    assertNotEquals(5, 2 + 2);

    # 3. True/False断言
    assertTrue(10 > 5);
    assertFalse(10 < 5);

    # 4. 空值断言
    String s = null;
    assertNull(s);

    String s2 = "hello";
    assertNotNull(s2);

    # 5. 数组断言
    assertArrayEquals(new int[]{1, 2, 3}, new int[]{1, 2, 3});

    # 6. 异常断言
    assertThrows(ArithmeticException.class, () -> {
        int x = 10 / 0;
    });

    # 7. 超时断言
    assertTimeout(Duration.ofSeconds(1), () -> {
        Thread.sleep(500);
    });

    # 8. 分组断言（全部执行，一起报告失败）
    assertAll("user",
        () -> assertEquals("John", user.getFirstName()),
        () -> assertEquals("Doe", user.getLastName()),
        () -> assertEquals(30, user.getAge())
    );
}
```

## 第三步：Mockito模拟

### 问题：如何测试依赖外部服务的代码？

```
场景：
OrderService依赖：
- OrderRepository（数据库）
- PaymentService（支付API）
- SmsService（短信API）

问题：
- 不希望真实调用数据库
- 不希望真实调用支付API
- 不希望真实发送短信

解决：
使用Mock对象模拟依赖
```

### 代码实现：Mock测试

```java
// service/OrderService.java
@Service
public class OrderService {

    @Autowired
    private OrderRepository orderRepository;

    @Autowired
    private PaymentService paymentService;

    @Autowired
    private SmsService smsService;

    public Order createOrder(CreateOrderRequest request) {
        # 创建订单
        Order order = new Order();
        order.setOrderNo(generateOrderNo());
        order.setUserId(request.getUserId());
        order.setTotalAmount(request.getAmount());

        # 保存订单
        Order saved = orderRepository.save(order);

        # 发送短信
        smsService.sendSms(order.getUserId(), "订单创建成功");

        return saved;
    }
}
```

```java
// test/service/OrderServiceTest.java
@SpringBootTest
class OrderServiceTest {

    @Autowired
    private OrderService orderService;

    @MockBean  # Spring Boot提供的Mock注解
    private OrderRepository orderRepository;

    @MockBean
    private SmsService smsService;

    @Test
    @DisplayName("创建订单成功")
    void testCreateOrder() {
        # given
        CreateOrderRequest request = new CreateOrderRequest();
        request.setUserId(1L);
        request.setAmount(new BigDecimal("99.99"));

        # 模拟Repository返回
        Order mockOrder = new Order();
        mockOrder.setId(1L);
        mockOrder.setOrderNo("TS20231201123456");
        when(orderRepository.save(any())).thenReturn(mockOrder);

        # when
        Order result = orderService.createOrder(request);

        # then
        assertNotNull(result);
        assertEquals("TS20231201123456", result.getOrderNo());

        # 验证Mock被调用
        verify(orderRepository).save(any());
        verify(smsService).sendSms(eq(1L), eq("订单创建成功"));
    }

    @Test
    @DisplayName("创建订单失败 - 金额为空")
    void testCreateOrderWithNullAmount() {
        # given
        CreateOrderRequest request = new CreateOrderRequest();
        request.setUserId(1L);
        request.setAmount(null);

        # when & then
        assertThrows(IllegalArgumentException.class, () -> {
            orderService.createOrder(request);
        });

        # 验证没有调用保存
        verify(orderRepository, never()).save(any());
    }
}
```

**Mockito常用方法**

```java
# 1. 模拟返回值
when(orderRepository.findById(1L)).thenReturn(Optional.of(order));

# 2. 模拟抛异常
when(orderRepository.findById(999L)).thenThrow(new RuntimeException("订单不存在"));

# 3. 模拟连续调用
when(orderRepository.findById(1L))
    .thenReturn(Optional.of(order1))
    .thenReturn(Optional.of(order2));

# 4. 验证调用
verify(orderRepository).findById(1L);
verify(orderRepository, times(2)).findById(1L);
verify(orderRepository, never()).delete(any());

# 5. 参数匹配
when(orderRepository.findById(any())).thenReturn(Optional.of(order));
when(orderRepository.findById(eq(1L))).thenReturn(Optional.of(order));
when(orderRepository.save(argThat(order -> order.getAmount() > 0))).thenReturn(order);

# 6.spy（部分模拟）
List list = new ArrayList();
List spyList = spy(list);

when(spyList.size()).thenReturn(100);  # mock size()
spyList.add("test");  # 调用真实方法
```

### 代码实现：MockMvc测试

```java
// test/controller/OrderControllerTest.java
@SpringBootTest
@AutoConfigureMockMvc  # 自动配置MockMvc
class OrderControllerTest {

    @Autowired
    private MockMvc mockMvc;

    @MockBean
    private OrderService orderService;

    @Autowired
    private ObjectMapper objectMapper;

    @Test
    @DisplayName("创建订单 - 成功")
    void testCreateOrder_Success() throws Exception {
        # given
        CreateOrderRequest request = new CreateOrderRequest();
        request.setUserId(1L);
        request.setAmount(new BigDecimal("99.99"));

        Order mockOrder = new Order();
        mockOrder.setId(1L);
        mockOrder.setOrderNo("TS20231201123456");

        when(orderService.createOrder(any())).thenReturn(mockOrder);

        # when & then
        mockMvc.perform(post("/api/order")
                .contentType(MediaType.APPLICATION_JSON)
                .content(objectMapper.writeValueAsString(request)))
                .andExpect(status().isOk())
                .andExpect(jsonPath("$.code").value(200))
                .andExpect(jsonPath("$.data.orderNo").value("TS20231201123456"));
    }

    @Test
    @DisplayName("创建订单 - 参数校验失败")
    void testCreateOrder_ValidationError() throws Exception {
        # given
        CreateOrderRequest request = new CreateOrderRequest();
        # amount为null，应该校验失败

        # when & then
        mockMvc.perform(post("/api/order")
                .contentType(MediaType.APPLICATION_JSON)
                .content(objectMapper.writeValueAsString(request)))
                .andExpect(status().isBadRequest());
    }

    @Test
    @DisplayName("获取订单 - 成功")
    void testGetOrder_Success() throws Exception {
        # given
        Order mockOrder = new Order();
        mockOrder.setId(1L);
        mockOrder.setOrderNo("TS20231201123456");

        when(orderService.getOrder(1L)).thenReturn(mockOrder);

        # when & then
        mockMvc.perform(get("/api/order/1"))
                .andExpect(status().isOk())
                .andExpect(jsonPath("$.data.id").value(1))
                .andExpect(jsonPath("$.data.orderNo").value("TS20231201123456"));
    }

    @Test
    @DisplayName("获取订单 - 不存在")
    void testGetOrder_NotFound() throws Exception {
        # given
        when(orderService.getOrder(999L))
            .thenThrow(new BusinessException("订单不存在"));

        # when & then
        mockMvc.perform(get("/api/order/999"))
                .andExpect(status().isOk())
                .andExpect(jsonPath("$.code").value(500))
                .andExpect(jsonPath("$.message").value("订单不存在"));
    }
}
```

## 第五步：Repository测试

### 问题：如何测试数据访问层？

```java
// test/repository/OrderRepositoryTest.java
@DataJpaTest  # 只测试JPA相关，不加载整个应用
class OrderRepositoryTest {

    @Autowired
    private TestEntityManager entityManager;  # 测试实体管理器

    @Autowired
    private OrderRepository orderRepository;

    @Test
    @DisplayName("根据订单号查询")
    void testFindByOrderNo() {
        # given
        Order order = new Order();
        order.setOrderNo("TS20231201123456");
        order.setUserId(1L);
        order.setStatus(0);
        entityManager.persist(order);  # 插入测试数据
        entityManager.flush();

        # when
        Optional<Order> result = orderRepository.findByOrderNo("TS20231201123456");

        # then
        assertTrue(result.isPresent());
        assertEquals("TS20231201123456", result.get().getOrderNo());
    }

    @Test
    @DisplayName("查询待支付订单")
    void testFindPendingOrders() {
        # given
        Order order1 = new Order();
        order1.setOrderNo("TS001");
        order1.setStatus(0);  # 待支付
        entityManager.persist(order1);

        Order order2 = new Order();
        order2.setOrderNo("TS002");
        order2.setStatus(1);  # 已支付
        entityManager.persist(order2);

        entityManager.flush();

        # when
        List<Order> pendingOrders = orderRepository.findByStatus(0);

        # then
        assertEquals(1, pendingOrders.size());
        assertEquals("TS001", pendingOrders.get(0).getOrderNo());
    }
}
```

## ⚠️ 测试最佳实践

### 1. 测试命名

```java
# 好的命名
void testCreateOrder_Success() {}
void testCreateOrder_WithNullAmount_ThrowsException() {}
void testGetOrder_NotFound() {}

# 不好的命名
void test1() {}
void testOrder() {}
```

### 2. AAA模式

```java
@Test
void test() {
    # Arrange（准备）
    Order order = new Order();
    order.setOrderNo("TS001");

    # Act（执行）
    Order saved = orderRepository.save(order);

    # Assert（断言）
    assertNotNull(saved.getId());
}
```

### 3. 独立性

```java
# 每个测试独立，不依赖执行顺序
# 不依赖共享状态
# 使用@BeforeEach初始化
```

### 4. 测试边界

```java
@Test
void testBoundary() {
    # 测试边界值
    assertEquals(0, calculator.divide(0, 1));
    assertEquals(Integer.MAX_VALUE, calculator.add(Integer.MAX_VALUE, 0));
}
```

## 📊 Django vs Spring Boot 对比

| 功能 | Django | Spring Boot | 难度 |
|------|--------|-------------|------|
| 测试框架 | pytest / unittest | JUnit 5 | 相当 |
| 断言 | assert | Assertions | Django更简洁 |
| Mock | unittest.mock | Mockito | 相当 |
| Web测试 | Django test client | MockMvc | 相当 |
| 数据库测试 | 测试数据库 | H2 / Testcontainers | 相当 |
| 覆盖率 | pytest-cov | JaCoCo | 相当 |

**Django优势**：
- pytest语法简洁
- 测试数据库自动管理

**Spring Boot优势**：
- @SpringBootTest功能强大
- MockMvc类型安全
- Testcontainers隔离性好
