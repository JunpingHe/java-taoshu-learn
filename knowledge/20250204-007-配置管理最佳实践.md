# 20250204-配置管理最佳实践

## 📋 知识点总结

### 1. 不推荐的做法

```java
// ❌ 到处写类加载器代码
public class Main {
    public void doSomething() {
        InputStream is = Main.class.getClassLoader()
            .getResourceAsStream("config.properties");
        // ...
    }
}

public class UserService {
    public void doSomething() {
        InputStream is = UserService.class.getClassLoader()
            .getResourceAsStream("config.properties");
        // ...
    }
}
```

**问题：**
- 代码重复
- 配置文件路径硬编码
- 维护困难（改配置文件名要改很多地方）
- 每次都重新加载，浪费资源

### 2. 推荐的做法：统一配置管理类

```java
/**
 * 配置管理工具类 - 单例模式
 */
public class ConfigManager {
    private static ConfigManager instance;
    private Properties props;

    // 私有构造函数
    private ConfigManager() {
        loadConfig();
    }

    // 获取单例实例
    public static ConfigManager getInstance() {
        if (instance == null) {
            synchronized (ConfigManager.class) {
                if (instance == null) {
                    instance = new ConfigManager();
                }
            }
        }
        return instance;
    }

    // 加载配置（只加载一次）
    private void loadConfig() {
        InputStream is = ConfigManager.class.getClassLoader()
            .getResourceAsStream("config.properties");
        props = new Properties();
        props.load(is);
    }

    // 获取配置值
    public String get(String key) {
        return props.getProperty(key);
    }

    // 便捷方法
    public String getAppName() {
        return get("app.name");
    }
}
```

### 3. 业务代码中使用

```java
// ✅ 简洁清晰
public class UserService {
    public void doSomething() {
        String dbUrl = ConfigManager.getInstance().get("db.url");
        String appName = ConfigManager.getInstance().getAppName();
        // 使用配置...
    }
}
```

### 4. 单例模式要点

| 要素 | 说明 |
|------|------|
| **私有构造函数** | 防止外部 new 创建实例 |
| **静态实例变量** | 全局唯一 |
| **公共静态方法** | 获取实例 |
| **双重检查锁** | 线程安全且高效 |
| **配置缓存** | 只加载一次，避免重复IO |

### 5. 最佳实践对比

| 方面 | 不推荐做法 | 推荐做法 |
|------|-----------|----------|
| **代码位置** | 分散在各个类中 | 集中在 ConfigManager |
| **配置加载** | 每次都加载 | 启动时加载一次，缓存使用 |
| **路径管理** | 硬编码在各处 | 统一在一处管理 |
| **维护性** | 改配置名要改多处 | 只需改 ConfigManager |

---

## 💡 关键要点

> 配置管理的最佳实践是创建单例的 ConfigManager 类，统一管理所有配置的加载和访问。这样可以避免代码重复、提高性能、便于维护。单例模式确保配置只加载一次，全局共享。
