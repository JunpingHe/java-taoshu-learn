# 20250204-JDK配置问题解决

## 📋 知识点总结

### 1. 常见错误：无法启动 javac 进程

```
错误：被配置为使用 JDK 0，但 IDE 仅支持使用 JDK 7 和更高版本
```

**原因分析：**
- IDEA 配置文件中指定了 JDK 版本（如 25）
- 但 IDEA 内部没有注册这个 JDK
- 导致显示为 "JDK 0"

### 2. 解决方案

#### 方案一：IDEA 自动下载（推荐）
```
弹窗提示：为缺失的 JDK"21"下载 Microsoft OpenJDK 21.0.10
      ↓
   点击"下载"
      ↓
IDEA 自动安装并配置
```

#### 方案二：手动配置
1. `Ctrl + Alt + Shift + S` 打开项目结构
2. **SDKs** → 添加 JDK → 选择路径
3. **项目** → 选择 SDK → 设置语言级别
4. **模块** → 设置语言级别

### 3. 配置文件说明

`.idea/misc.xml`:
```xml
<component name="ProjectRootManager"
    version="2"
    project-jdk-name="21"        ← 使用的JDK版本
    project-jdk-type="JavaSDK">
    <output url="file://$PROJECT_DIR$/out" />
</component>
```

`pom.xml`:
```xml
<maven.compiler.source>21</maven.compiler.source>
<maven.compiler.target>21</maven.compiler.target>
```

### 4. 三级配置关系

```
SDKs（全局）
  ↓
项目级别（Project）
  ↓
模块级别（Module）
```

---

## 💡 关键要点

> 遇到JDK配置问题，优先使用IDEA的自动下载功能。如果手动配置，记住三级配置：SDKs → 项目 → 模块，保持版本一致。
