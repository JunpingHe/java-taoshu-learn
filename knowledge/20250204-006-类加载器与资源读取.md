# 20250204-类加载器与资源读取

## 📋 知识点总结

### 1. 读取 resources 的代码解析

```java
Main.class.getClassLoader().getResourceAsStream("config.properties")
   ↓        ↓              ↓                           ↓
  类名  获取Class对象   获取类加载器              读取资源为流
```

| 代码 | 方法定义在 | 作用 |
|------|-----------|------|
| `Main.class` | Java 语言关键字 | 获取该类的 `Class` 对象 |
| `.getClassLoader()` | `java.lang.Class` 类 | 获取加载该类的类加载器 |
| `.getResourceAsStream()` | `java.lang.ClassLoader` 类 | 从 classpath 读取资源 |

### 2. 类继承关系

```
        Object (所有类的父类)
           ↑
           │ 继承
           │
        Main (你的类)
           │
           │ .class (获取 Class 对象)
           ↓
     Class<Main> (描述 Main 类的元信息)
           │
           │ .getClassLoader() (获取类加载器)
           ↓
     ClassLoader (负责加载类的工具)
           │
           │ .getResourceAsStream() (读取资源)
           ↓
     InputStream (资源内容)
```

### 3. 类加载器的重要事实

**真相：同一应用中，所有类共享同一个类加载器！**

```java
// 验证代码
ClassLoader loader1 = Main.class.getClassLoader();
ClassLoader loader2 = UserService.class.getClassLoader();
ClassLoader loader3 = ConfigManager.class.getClassLoader();

// 结果：loader1 == loader2 == loader3 (同一个对象！)
```

### 4. 为什么用任意类都能读取配置？

```
┌─────────────────────────────────────┐
│  AppClassLoader (唯一的类加载器)     │
│  ┌─────────────────────────────┐   │
│  │  类加载器指向 classpath      │   │
│  └─────────────────────────────┘   │
│           ↑        ↑        ↑       │
│        Main.class  UserService   ConfigManager
│                                       │
└─────────────────────────────────────┘

因为类加载器是同一个，所以都能读取 classpath 下的资源
```

### 5. 读取资源的方式对比

| 做法 | 代码示例 | 评价 |
|------|----------|------|
| ❌ 绝对路径 | `new File("D:/project/config.properties")` | 路径硬编码，换电脑失效 |
| ❌ 相对路径 | `new File("src/main/resources/...")` | 打包后 src 目录不存在 |
| ✅ 类加载器 | `XXX.class.getClassLoader().getResourceAsStream(...)` | 开发和打包都可用 |

---

## 💡 关键要点

> 同一应用中所有类的类加载器是同一个对象（AppClassLoader）。因此用任意类获取类加载器都能读取到相同的 classpath 资源。这不是"每个类的类加载器"，而是"共享的类加载器"。
