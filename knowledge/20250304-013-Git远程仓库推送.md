# Git远程仓库推送

**日期**：2025-03-04

## 问题
如何将本地项目推送到 GitHub？

## 解决方案

### 方式一：使用 GitHub CLI（推荐）
```bash
gh repo create 仓库名 --public --source=. --remote=origin --push
```

### 方式二：手动创建后推送

1. 在 GitHub 网站创建仓库（不初始化 README）

2. 添加远程仓库：
```bash
git remote add origin https://github.com/用户名/仓库名.git
```

3. 推送代码：
```bash
git push -u origin master
```

## 常用命令

| 命令 | 说明 |
|------|------|
| `git remote -v` | 查看远程仓库地址 |
| `git remote add origin <url>` | 添加远程仓库 |
| `git push -u origin branch` | 推送并设置上游分支 |
| `git push` | 推送到已设置的上游分支 |

## 注意事项

- Windows 保留名称（如 `nul`）不能用作文件名
- LF/CRLF 警告是正常的，Git 会自动处理换行符转换
- 首次推送到空仓库需要使用 `-u` 参数

## 国内网络问题

如果 HTTPS 连接失败，可使用 SSH 方式：

```bash
# 1. 测试 SSH 连接
ssh -T git@github.com

# 2. 切换远程地址为 SSH
git remote remove origin
git remote add origin git@github.com:用户名/仓库名.git

# 3. 推送
git push -u origin master
```

**SSH 优势**：无需输入密码，更安全可靠
