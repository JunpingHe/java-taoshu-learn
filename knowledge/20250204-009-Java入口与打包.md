# Java程序入口与打包

## 📅 日期：2025-02-04

## 🎯 核心问题
一个项目有多个类都有main方法，JVM运行哪个？

## 💡 核心理解

### 1. 编译 vs 运行

| 阶段 | 命令 | 是否需要入口 | 说明 |
|------|------|-------------|------|
| **编译** | `mvn compile` | ❌ 不需要 | 编译所有.java文件，不运行 |
| **运行** | `mvn exec:java` | ✅ 必须指定 | JVM需要知道从哪开始 |

### 2. 运行时指定入口

```bash
# 方式一：Maven命令
mvn exec:java -Dexec.mainClass="org.example.HelloJava"

# 方式二：java命令
java -cp target/classes org.example.HelloJava

# 方式三：IDEA中
点击类旁边的绿色 ▶️ 按钮
```

### 3. 打包成jar（推荐）

#### 配置 pom.xml

```xml
<build>
    <plugins>
        <plugin>
            <groupId>org.apache.maven.plugins</groupId>
            <artifactId>maven-jar-plugin</artifactId>
            <version>3.3.0</version>
            <configuration>
                <archive>
                    <manifest>
                        <!-- 🎯 打包时指定入口类 -->
                        <mainClass>org.example.HelloJava</mainClass>
                    </manifest>
                </archive>
            </configuration>
        </plugin>
    </plugins>
</build>
```

#### 打包流程

```
mvn package
    ↓
生成 jar 包
    ↓
主类写入 META-INF/MANIFEST.MF
    ↓
Main-Class: org.example.HelloJava
    ↓
java -jar app.jar
    ↓
JVM自动读取MANIFEST.MF找到入口！
```

#### 运行jar包

```bash
# 不需要指定类名！
java -jar target/taoshu-learn-1.0-SNAPSHOT.jar
```

## 🔑 关键概念

### 一个项目可以有多个main方法吗？

**答案：完全可以！**

```java
// 类A
public class A {
    public static void main(String[] args) {
        System.out.println("我是A");
    }
}

// 类B
public class B {
    public static void main(String[] args) {
        System.out.println("我是B");
    }
}

// 类C
public class C {
    public static void main(String[] args) {
        System.out.println("我是C");
    }
}
```

你想运行哪个，就告诉JVM运行哪个！

### 为什么要这样设计？

1. **方便测试** - 每个类可以独立测试
2. **灵活启动** - 一个程序多个入口点
3. **工具类** - 可以单独运行验证功能

## 📋 常用命令

```bash
# 编译
mvn compile

# 运行（指定类）
mvn exec:java -Dexec.mainClass="org.example.HelloJava"

# 打包
mvn package

# 运行jar包
java -jar target/taoshu-learn-1.0-SNAPSHOT.jar

# 查看jar包的MANIFEST.MF
jar xf app.jar META-INF/MANIFEST.MF
cat META-INF/MANIFEST.MF
```

## ⚠️ 注意事项

1. **编译时不需要入口** - 只有运行时才需要
2. **pom.xml配置的是打包入口** - 只对 `java -jar` 有效
3. **Spring Boot也是同样原理** - 只是配置方式不同

## 🎓 知识延伸

### Spring Boot的启动方式

```java
@SpringBootApplication
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}
```

- 开发时：IDEA运行或 `mvn spring-boot:run`
- 打包后：`java -jar app.jar`（MANIFEST.MF已记录主类）
