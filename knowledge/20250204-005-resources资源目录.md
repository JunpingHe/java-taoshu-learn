# 20250204-resources资源目录

## 📋 知识点总结

### 1. resources 目录不会自动创建

```
需要手动创建：
mkdir -p src/main/resources
```

### 2. resources 目录存放内容

| 类型 | 目录/文件 | 用途 | 示例 |
|------|-----------|------|------|
| **配置文件** | `*.properties` | 应用配置 | `config.properties` |
| **日志配置** | `logback.xml` | 日志框架配置 | 输出格式、级别 |
| **静态资源** | `static/` | 前端静态文件 | HTML/CSS/JS/图片 |
| **模板文件** | `templates/` | 动态页面模板 | Thymeleaf 模板 |
| **国际化** | `i18n/` | 多语言文件 | `messages_zh_CN.properties` |

### 3. resources 目录结构

```
src/main/resources/
├── config.properties      # 配置文件（键值对）
├── logback.xml            # 日志配置
├── static/                # 静态资源
│   └── index.html        # HTML 文件
├── templates/             # 模板文件
└── i18n/                  # 国际化文件
```

### 4. 为什么放在 resources？

```
❌ 错误：放在项目根目录
taoshu-learn/config.properties    ← 打包时丢失

✅ 正确：放在 resources
src/main/resources/config.properties    ← 自动打包到 jar
```

**原因：**
- Maven 打包时自动包含 resources 内容
- Java 可以用 `getClass().getResource()` 读取
- 统一管理所有非代码资源

### 5. Maven 打包后的结构

```
taoshu-learn.jar
├── org/example/Main.class       ← Java 字节码
├── org/example/ConfigManager.class
└── config.properties            ← resources 里的文件
```

---

## 💡 关键要点

> resources 目录是 Maven 约定的资源目录，不会自动创建，需要手动建立。所有配置文件、静态资源都放在这里，打包时会自动包含到 jar 中。
