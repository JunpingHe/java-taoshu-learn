# 20250204-Maven项目结构

## 📋 知识点总结

### 1. Maven 项目标准结构

```
taoshu-learn/
├── pom.xml                 # Maven配置文件（项目身份证）
├── src/                    # 源代码目录
│   └── main/
│       ├── java/          # Java源代码
│       │   └── org/example/Main.java
│       └── resources/     # 配置文件
└── target/                # 编译产物（自动生成）
    └── classes/           # .class字节码文件
```

### 2. pom.xml 核心配置

```xml
<groupId>org.example</groupId>      # 组织域名倒写（全球唯一）
<artifactId>taoshu-learn</artifactId> # 项目名称
<version>1.0-SNAPSHOT</version>     # 版本号
```

### 3. 常用 Maven 命令

| 命令 | 作用 |
|------|------|
| `mvn compile` | 编译源代码 |
| `mvn exec:java -Dexec.mainClass="xxx"` | 运行主类 |
| `mvn clean package` | 清理并打包 |
| `mvn test` | 运行测试 |

### 4. 为什么这样设计？

- **约定优于配置**：固定目录结构，无需额外配置
- **坐标体系**：groupId + artifactId + version 唯一定位项目
- **自动化构建**：target目录可随时重建，不纳入版本控制

---

## 💡 关键要点

> Maven的核心思想是"约定优于配置"——所有项目遵循相同结构，降低学习成本和团队协作成本。
