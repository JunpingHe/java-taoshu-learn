# 20250204-JDK版本选择

## 📋 知识点总结

### 1. JDK 版本发布规则

```
Java 17 (LTS) → 18 → 19 → 20 → Java 21 (LTS) → 22 → 23 → 24 → 25 (LTS)
               非LTS  非LTS  非LTS      稳定版      非LTS 非LTS  非LTS  最新
```

- **LTS** = Long Term Support（长期支持版本）
- LTS 版本：Java 8, 11, 17, 21, 25
- 非LTS版本：每6个月发布一次，只支持6个月

### 2. IDEA 与 JDK 版本匹配

| IDEA 版本 | 最高支持 JDK |
|-----------|--------------|
| 2024.1.x | Java 22 |
| 2024.2.x | Java 23 |
| 2024.3.x | Java 24 |
| 2025.1.x | Java 25 |

### 3. 多版本 JDK 共存

```
系统配置：
JAVA_HOME → JDK 25（命令行 mvn、java 使用）

IDEA 项目配置：
.project-jdk-name="21" → JDK 21（IDEA 内部使用）
```

**为什么可以共存？**
- 每个 JDK 是独立文件夹
- 项目配置指定使用哪个版本
- 互不干扰，各司其职

### 4. 版本选择建议

| 场景 | 推荐版本 | 原因 |
|------|----------|------|
| **学习** | JDK 21 | LTS稳定，IDEA完美支持 |
| **新特性探索** | JDK 25 | 最新预览功能 |
| **生产环境** | JDK 17/21 | LTS长期支持 |

---

## 💡 关键要点

> JDK选择三要素：IDEA支持度 + 稳定性（LTS优先） + 项目需求。多版本共存是常态，不必纠结"只留一个"。
